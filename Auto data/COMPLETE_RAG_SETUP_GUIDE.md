# 🚀 完整 RAG 知识库集成指南

## 📋 概述

这个指南将帮你设置一个完整的 RAG（检索增强生成）系统，让 AI Agent 真正访问你的 Google Docs 知识库。

## 🔧 工作流结构

```
Telegram Trigger → 获取知识库内容 → 解析知识库 → AI Agent (带知识库) → 处理AI输出 → 执行查询 → 返回结果
```

## 📝 配置步骤

### **第1步：导入工作流**

1. 在 n8n 中导入 `complete-rag-query.json`
2. 确保所有节点正确连接

### **第2步：配置 Telegram Bot**

1. **Telegram Trigger 节点**：
   - 确保 Telegram Bot Token 已配置
   - 测试消息接收

### **第3步：配置知识库获取**

1. **获取知识库内容 节点**：
   - URL: `https://docs.google.com/document/d/187GD-WKLn_EzyO_eEGp-jP0HgKJlQ53_QOrvbdrk1e0/export?format=txt`
   - 这个 URL 会自动从你的 Google Docs 获取纯文本内容

2. **解析知识库 节点**：
   - 自动解析 Markdown 格式
   - 提取各个场景案例
   - 生成知识库内容字符串

### **第4步：配置 AI Agent**

1. **AI Agent (带知识库) 节点**：
   - 系统消息包含完整的数据库结构
   - **关键**：包含 `{{ $json.knowledgeBaseContent }}` 引用
   - 强制中文输出
   - 参考知识库案例生成 SQL

### **第5步：配置查询执行**

1. **Start Query 节点**：
   - API URL: `https://organ-execute-contract-coaches.trycloudflare.com/api/async/start`
   - API Key: 你的后端 API Key
   - 自动发送 SQL 和数据库参数

2. **Check Status 节点**：
   - 轮询查询状态
   - 等待查询完成

### **第6步：配置结果处理**

1. **Process Result 节点**：
   - 处理查询结果
   - 格式化输出数据

2. **Send Telegram Result 节点**：
   - 发送格式化的查询结果
   - 包含知识库使用状态

## 🎯 关键特性

### **✅ 真正的知识库集成**
- 每次查询都会从 Google Docs 获取最新知识库内容
- AI Agent 系统消息包含完整的知识库案例

### **✅ 完整的错误处理**
- 查询失败时的详细错误信息

## 🚨 故障排除

### **问题1：知识库内容为空**
- 检查 Google Docs 链接是否正确
- 确保文档是公开可访问的
- 验证导出格式 URL

### **问题2：AI Agent 仍输出英文**
- 检查系统消息中的中文要求
- 尝试使用不同的 AI 模型
- 重启 n8n 服务

### **问题3：查询执行失败**
- 检查后端服务是否运行
- 验证 API Key 是否正确
- 检查 Cloudflare Tunnel 状态
