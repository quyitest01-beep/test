# 阶段一：智能查询（AI + Athena）- 里程碑

## 📋 阶段目标

实现自然语言输入 → 自动生成SQL → 执行Athena查询 → 支持大数据导出

---

## 🎯 核心功能

### 1. 自然语言转SQL
- 支持中文/英文自然语言查询
- 智能识别查询意图（统计/筛选/排序/聚合）
- 自动提取查询参数（时间范围/条件/字段）
- 生成可执行的标准SQL语句

### 2. SQL执行与查询结果展示
- 连接AWS Athena执行查询
- 实时显示查询进度和状态
- 格式化展示查询结果
- 显示执行时间、扫描数据量、查询成本

### 3. 数据清洗、分页与导出
- 自动去重、空值处理
- 支持结果分页显示
- 导出Excel/CSV格式
- 大数据集自动拆分（>10万行）

## 📝 测试用例

### 用例1：简单统计查询
**输入**：
```
查询最近7天的用户数
```

**预期SQL**：
```sql
SELECT COUNT(DISTINCT user_id) as user_count
FROM users
WHERE DATE(created_at) >= DATE_SUB(CURRENT_DATE, INTERVAL 7 DAY)
```

**预期输出**：
- 返回用户数统计结果
- 生成Excel文件
- 响应时间 < 10秒

---

### 用例2：多条件筛选
**输入**：
```
查询10月1日到10月7日，PHP和INR货币的充值金额和笔数
```

**预期SQL**：
```sql
SELECT 
  currency,
  COUNT(*) as transaction_count,
  SUM(amount) as total_amount
FROM transactions
WHERE DATE(transaction_date) BETWEEN DATE '2024-10-01' AND DATE '2024-10-07'
  AND currency IN ('PHP', 'INR')
  AND status = 'success'
GROUP BY currency
```

**预期输出**：
- 返回分币种的统计数据
- 包含笔数和金额
- Excel格式下载

## 🐛 风险与应对

### 风险1：自然语言理解不准确
**影响**：生成的SQL不符合预期
**概率**：中
**应对措施**：
- 提供SQL预览，用户可确认后执行
- 支持用户直接输入SQL绕过AI
- 积累常用查询模板
- 持续优化AI提示词

### 风险2：查询超时
**影响**：大数据查询失败
**概率**：中
**应对措施**：
- 设置合理的超时时间（默认120秒）
- 自动优化SQL（添加LIMIT）
- 提示用户优化查询条件
- 支持异步查询（后台执行，完成后通知）

### 风险3：导出数据量过大
**影响**：Excel生成失败或下载缓慢
**概率**：高
**应对措施**：
- 限制单次查询最大行数（50万）
- 自动分批导出
- 提供CSV格式选项（更轻量）
- 实现流式导出

### 风险4：成本超标
**影响**：AWS费用过高
**概率**：低
**应对措施**：
- 设置单次查询成本上限（$1）
- 实施查询结果缓存
- 优化SQL减少数据扫描
- 监控每日成本，超标告警
