# 周度/月度数据报告生成工作流使用说明

## 工作流概述

本工作流参考「游戏评级报告生成工作流」设计，用于自动生成**周度/月度数据报告**。

### 核心功能

1. **数据获取**：从Lark邮件自动下载营收数据和用户数据附件
2. **数据聚合**：按游戏、商户、货币维度聚合数据
3. **趋势分析**：计算环比/同比指标，识别异常
4. **报告生成**：AI生成结构化Markdown报告
5. **格式转换**：Markdown → HTML → PDF
6. **自动上传**：PDF上传到Lark多维表
7. **通知推送**：发送Lark卡片通知

---

## 工作流结构

```
手动触发
  ↓
环境变量设置（周度/月度模式）
  ↓
获取Lark Access Token
  ↓
【并行分支1：营收数据】          【并行分支2：用户数据】
  ↓                                    ↓
列出邮件（收件箱/发件箱）           列出邮件（收件箱/发件箱）
  ↓                                    ↓
筛选目标邮件-营收                   筛选目标邮件-用户
  ↓                                    ↓
下载附件 & 提取表格数据             下载附件 & 提取表格数据
  ↓                                    ↓
        └────────── Merge ──────────┘
                      ↓
        数据聚合（周度/月度）
                      ↓
        计算趋势（环比/同比 + 异常检测）
                      ↓
        AI生成报告（Markdown）
                      ↓
        Markdown → HTML
                      ↓
        HTML → PDF
                      ↓
        上传PDF到Lark
                      ↓
        发送Lark通知
```

## 关键节点说明

### 1. 环境变量设置

**功能**：根据模式（weekly/monthly）计算目标周期

**输入**：
```json
{
  "mode": "weekly" | "monthly"
}
```

**输出**：
```json
{
  "report_type": "weekly" | "monthly",
  "target_period": "20251110-20251116" | "2025年11月",
  "target_yyyy_mm": "202511"
}
```

---

### 2. 筛选目标邮件-营收

**功能**：根据报告类型筛选营收数据邮件

**周度模式**：
- 主题：`【即时】周度详细汇总报表 - {YYYYMMDD-YYYYMMDD}`
- 附件：`{YYYYMMDD}_merchant_currency_game.xlsx`
- 发件人：`billing@gaming-panda.com`

**月度模式**：
- 主题：`【即时】月度详细汇总报表 - {YYYY年MM月}`
- 附件：`{YYYYMM}_merchant_currency_game.xlsx`
- 发件人：`billing@gaming-panda.com`

---

### 3. 筛选目标邮件-用户

**功能**：根据报告类型筛选用户数据邮件

**周度模式**：
- 主题：`【{YYYYMMDD-YYYYMMDD}】周度用户数据报表`
- 附件：`游戏新用户留存数据.xlsx`
- 发件人：`poon@gaming-panda.com`

**月度模式**：
- 主题：`【{YYYY年MM月}】月度用户数据报表` 或 `【{MM}月】月度用户数据报表`
- 附件：`游戏新用户留存数据.xlsx`
- 发件人：`poon@gaming-panda.com`

---

**功能**：
- 聚合营收数据：按游戏、商户、货币汇总
- 聚合用户数据：按游戏、商户计算留存率
- 计算全局汇总指标

**输出结构**：
```json
{
  "report_type": "weekly" | "monthly",
  "period": "20251110-20251116" | "2025年11月",
  "global_summary": {

    "total_payout_usd": 2300000,
    "total_ggr_usd": 200000,
    "avg_d1_retention": 5.2,
    "avg_d7_retention": 2.1,
    "avg_ggr_per_user": 1.59
  },
  "games": [
    {
      "game_name": "Super Ace",
      "total_new_users": 32556,
      "total_ggr_usd": 216000,
      "avg_d1_retention": 9.0,
      "avg_d7_retention": 3.0,
      "merchants": [...]
    }
  ],
  "merchants": [...]
}
```

---

### 5. 计算趋势（`weekly-monthly-trend-calculator.js`）

**功能**：
- 计算环比/同比指标
- 识别异常（负GGR、低留存、低用户数）
- 生成Top 10排名

**输出结构**：
```json
{
  "trends": {
    "global": {
      "total_new_users": {
        "current": 125430,
        "previous": 109000,
        "growth_rate": 15.07,
        "trend": "📈"
      }
    }
  },
  "rankings": {
    "top10_games": [...],
    "top10_merchants": [...]
  },
  "anomalies": {
    "games": [
      {
        "game_name": "Monopoly",
        "issues": ["负GGR", "低留存率"],
        "severity": "P0"
      }
    ],
    "merchants": [...]
  }

```

---

### 6. AI生成报告

**功能**：根据聚合数据和趋势数据，生成结构化Markdown报告

**提示词**：参考 `weekly-monthly-report-ai-prompt.md`

**输出**：Markdown格式报告，包含：
1. 执行摘要（核心指标卡片 + 一句话结论 + 风险提示）
2. 核心发现（3-5个）
3. 分维度分析（Top 10游戏 + Top 10商户）
4. 趋势分析（仅月度报告）
5. 异常监控
6. 行动建议

---

### 7. 生成HTML

**功能**：将Markdown转换为彩色HTML

**参考**：游戏评级工作流的 `生成html-中` 节点

**输出**：完整HTML文档，包含样式和图表

---

### 8. 生成PDF

**功能**：将HTML转换为PDF

**API**：`https://sticky-dubai-gazette-towns.trycloudflare.com/render`

**输出**：PDF二进制数据

---

### 9. 上传PDF

**功能**：上传PDF到Lark多维表

**API**：`POST /open-apis/drive/v1/medias/upload_all`

**输出**：`file_token`

---

### 10. 整合Lark卡片 & 发送通知

**功能**：构建Lark交互式卡片并发送通知

**卡片内容**：
- 报告类型和周期
- 核心指标摘要
- 链接到PDF报告

## 报告结构说明

### 周度报告（2-3页）

**重点**：变化与异常

1. **执行摘要**：核心指标 + 环比 + 风险提示
2. **核心发现**：3-5个关键发现
3. **分维度分析**：Top 10游戏 + Top 10商户
4. **异常监控**：异常列表
5. **行动建议**：优先级排序

### 月度报告（5-8页）

**重点**：趋势与策略

1. **执行摘要**：核心指标 + 环比/同比 + 风险提示
2. **核心发现**：3-5个关键发现
3. **分维度分析**：Top 10游戏 + Top 10商户
4. **趋势分析**：时间序列图表 + 趋势判断
5. **异常监控**：异常列表
6. **行动建议**：优先级排序

## 使用步骤

### 1. 导入工作流

1. 打开n8n
2. 导入 `weekly-monthly-report-workflow.json`
3. 检查节点配置

### 2. 配置节点

**必须配置**：
- `获取Lark Access Token`：Lark应用凭证
- `Google Gemini Chat Model`：AI模型凭证
- `发送Lark通知`：接收消息的chat_id

**可选配置**：
- `环境变量设置`：默认模式（weekly/monthly）

### 3. 运行工作流

**方式1：手动触发**
- 点击「手动触发」节点
- 在输入JSON中指定 `mode: "weekly"` 或 `mode: "monthly"`

**方式2：定时触发**
- 添加 `Schedule Trigger` 节点
- 设置周度：每周一上午9点
- 设置月度：每月1日上午9点

### 4. 查看结果

- **Execution日志**：查看每个节点的执行情况
- **Lark通知**：收到卡片消息，包含报告链接
- **Lark多维表**：PDF已上传，可通过链接查看

## 注意事项

### 1. 数据依赖

- **营收数据**：必须来自 `billing@gaming-panda.com` 的月度/周度报表邮件
- **用户数据**：必须来自 `poon@gaming-panda.com` 的月度/周度用户数据报表邮件
- **历史数据**：如需计算环比/同比，需要提供上一周期的数据

### 2. 邮件格式

- 邮件主题必须严格匹配筛选条件
- 附件文件名必须包含指定格式的日期

### 3. 异常处理

- 如果未找到目标邮件，工作流会报错
- 如果数据格式不正确，聚合节点会跳过无效数据
- 如果AI生成失败，检查提示词和输入数据格式

### 4. 性能优化

- 周度报告：处理速度快，适合频繁查看
- 月度报告：数据量大，处理时间较长（建议在非高峰时段运行）

## 扩展功能

### 1. 添加历史数据对比

在「计算趋势」节点前，添加「获取历史数据」节点：
- 从Lark多维表读取上一周期数据
- 或从数据库查询历史数据

### 2. 添加图表生成

在「生成HTML」节点中：
- 使用Chart.js生成时间序列图表
- 使用D3.js生成饼图/柱状图

### 3. 添加多语言支持

- 创建英文版AI提示词
- 创建英文版HTML模板
- 并行生成中英文报告

## 故障排查

### 问题1：未找到目标邮件

**原因**：邮件主题或附件名不匹配

**解决**：
1. 检查邮件筛选节点的筛选条件
2. 确认邮件主题格式是否正确
3. 确认附件文件名格式是否正确

### 问题2：数据聚合失败

**原因**：数据格式不正确

**解决**：
1. 检查Excel文件结构
2. 确认字段名是否匹配
3. 查看节点日志，定位具体错误

### 问题3：AI生成报告格式错误

**原因**：提示词或输入数据格式问题

**解决**：
1. 检查AI提示词格式
2. 确认输入JSON结构是否正确
3. 查看AI节点输出，检查Markdown格式
