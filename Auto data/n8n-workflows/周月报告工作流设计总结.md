# 周度/月度数据报告工作流设计总结

## 设计思路

参考「游戏评级报告生成工作流」，设计一个**周度/月度数据报告生成工作流**，核心差异：

| 维度 | 游戏评级报告 | 周/月数据报告 |
|------|------------|--------------|
| **数据来源** | 游戏指标数据（留存、营收、RTP） | 用户数据 + 营收数据（Excel附件） |
| **分析维度** | 游戏维度（单个游戏评分） | 全局维度（平台整体表现） |
| **报告重点** | 游戏评级（S/A/B/C） | 数据趋势（环比/同比） |
| **报告长度** | 单游戏1页 | 周度2-3页，月度5-8页 |
| **输出格式** | 每个游戏一个PDF | 整个周期一个PDF |

---

## 工作流对比

### 游戏评级报告工作流
```
触发 → 环境变量 → Token → 邮件筛选 → 下载附件 → 提取数据 → 
计算评分 → AI生成报告 → HTML → PDF → 上传 → 通知
```

### 周/月数据报告工作流
```
触发 → 环境变量 → Token → 邮件筛选（并行：营收+用户） → 
下载附件 → 提取数据 → 数据聚合 → 计算趋势 → 
AI生成报告 → HTML → PDF → 上传 → 通知
```

**关键差异**：
1. **并行数据获取**：营收数据和用户数据并行处理
2. **数据聚合**：需要合并多个游戏/商户的数据
3. **趋势计算**：需要计算环比/同比，识别异常
4. **报告结构**：更复杂的报告结构（执行摘要、核心发现、趋势分析等）

---

## 关键节点设计

### 1. 环境变量设置

**功能**：根据模式（weekly/monthly）计算目标周期

**代码位置**：工作流JSON中的 `环境变量设置` 节点

**关键逻辑**：
- 周度：计算上周一至上周日
- 月度：计算上个月（YYYY年MM月）

---

### 2. 数据聚合节点

**文件**：`weekly-monthly-data-aggregator.js`

**功能**：
- 聚合营收数据：按游戏、商户、货币汇总
- 聚合用户数据：按游戏、商户计算留存率
- 计算全局汇总指标

**输入**：
- 营收数据：`{ "游戏名": "...", "汇总": {...}, "商户数据": [...] }`
- 用户数据：`{ "游戏名": "...", "汇总": {...}, "商户数据": [...] }`

**输出**：
```json
{
  "report_type": "weekly" | "monthly",
  "period": "...",
  "global_summary": {...},
  "games": [...],
  "merchants": [...]
}
```

---

### 3. 趋势计算节点

**文件**：`weekly-monthly-trend-calculator.js`

**功能**：
- 计算环比/同比指标
- 识别异常（负GGR、低留存、低用户数）
- 生成Top 10排名

**输入**：
- 当前周期数据
- 上一周期数据（可选）

**输出**：
```json
{
  "trends": {
    "global": {
      "total_new_users": {
        "current": 125430,
        "previous": 109000,
        "growth_rate": 15.07,
        "trend": "📈"
      }
    }
  },
  "rankings": {
    "top10_games": [...],
    "top10_merchants": [...]
  },
  "anomalies": {
    "games": [...],
    "merchants": [...]
  }
}
```

---

### 4. AI报告生成

**提示词文件**：`weekly-monthly-report-ai-prompt.md`

**报告结构**：
1. 执行摘要（核心指标卡片 + 一句话结论 + 风险提示）
2. 核心发现（3-5个）
3. 分维度分析（Top 10游戏 + Top 10商户）
4. 趋势分析（仅月度报告）
5. 异常监控
6. 行动建议

---

## 报告结构设计

### 周度报告（2-3页）

**重点**：变化与异常

1. **执行摘要**（1页）
   - 核心指标卡片（5个关键指标）
   - 一句话结论
   - 风险提示（Top 3）

2. **核心发现**（0.5页）
   - 3-5个关键发现
   - 每个发现：标题 + 数据 + 分析

3. **分维度分析**（1页）
   - Top 10游戏表格 + 分析
   - Top 10商户表格 + 分析

4. **异常监控**（0.5页）
   - 异常列表表格

5. **行动建议**（0.5页）
   - P0/P1/P2优先级排序

---

### 月度报告（5-8页）

**重点**：趋势与策略

1. **执行摘要**（1页）
   - 核心指标卡片（5个关键指标）
   - 一句话结论
   - 风险提示（Top 3）

2. **核心发现**（1页）
   - 3-5个关键发现
   - 每个发现：标题 + 数据 + 图表描述 + 分析

3. **分维度分析**（2页）
   - Top 10游戏表格 + 分析
   - Top 10商户表格 + 分析

4. **趋势分析**（2页）
   - 时间序列图表（4个）
   - 趋势判断
   - 拐点识别
   - 季节性/周期性

5. **异常监控**（1页）
   - 异常列表表格

6. **行动建议**（1页）
   - P0/P1/P2优先级排序

---

## 数据流设计

### 输入数据

**营收数据**（Excel）：
- 格式：`{ "游戏名": "...", "时间": "...", "汇总": {...}, "商户数据": [...] }`
- 来源：`billing@gaming-panda.com` 的月度/周度报表邮件
- 附件：`{YYYYMM}_merchant_currency_game.xlsx` 或 `{YYYYMMDD}_merchant_currency_game.xlsx`

**用户数据**（Excel）：
- 格式：`{ "游戏名": "...", "日期": "...", "当日用户数": ..., "次日用户数": ..., ... }`
- 来源：`poon@gaming-panda.com` 的月度/周度用户数据报表邮件
- 附件：`游戏新用户留存数据.xlsx`

---

### 数据处理流程

1. **下载附件**：从Lark邮件下载Excel附件
2. **提取表格数据**：使用 `Extract From File` 节点提取Excel数据
3. **数据聚合**：
   - 营收数据：按游戏、商户、货币聚合（参考 `处理数据-营收` 节点）
   - 用户数据：按游戏、商户计算留存率（参考 `处理数据-用户` 节点）
4. **合并数据**：使用 `Merge` 节点合并营收和用户数据
5. **计算趋势**：计算环比/同比，识别异常，生成Top排名
6. **AI生成报告**：根据聚合数据和趋势数据生成Markdown报告
7. **格式转换**：Markdown → HTML → PDF
8. **上传和通知**：上传PDF到Lark，发送通知

## 使用建议

### 1. 首次使用

1. 导入 `weekly-monthly-report-workflow.json` 到n8n
2. 配置节点：
   - `获取Lark Access Token`：Lark应用凭证
   - `Google Gemini Chat Model`：AI模型凭证
   - `发送Lark通知`：接收消息的chat_id
3. 测试运行：
   - 手动触发，输入 `{ "mode": "monthly" }`
   - 检查每个节点的输出
   - 确认报告格式正确

### 2. 定时运行

**周度报告**：
- 添加 `Schedule Trigger` 节点
- 设置：每周一上午9点
- 输入：`{ "mode": "weekly" }`

**月度报告**：
- 添加 `Schedule Trigger` 节点
- 设置：每月1日上午9点
- 输入：`{ "mode": "monthly" }`

### 3. 历史数据对比

如需计算环比/同比，需要：
1. 在「计算趋势」节点前，添加「获取历史数据」节点
2. 从Lark多维表或数据库读取上一周期数据
3. 将历史数据合并到当前数据中

## 优化建议

### 1. 性能优化

- **并行处理**：营收数据和用户数据并行下载和处理
- **批量操作**：使用批量API减少HTTP请求次数
- **缓存机制**：缓存历史数据，避免重复查询

### 2. 功能扩展

- **多语言支持**：生成中英文双语报告
- **图表生成**：使用Chart.js生成时间序列图表
- **自定义报告**：支持按角色定制报告内容

### 3. 错误处理

- **数据验证**：在数据聚合节点添加数据格式验证
- **异常重试**：在HTTP请求节点添加重试机制
- **日志记录**：记录关键节点的执行日志

---

## 注意事项

1. **邮件格式**：确保邮件主题和附件名格式正确
2. **数据格式**：确保Excel文件结构符合预期
3. **历史数据**：如需环比/同比，需要提供上一周期数据
4. **AI提示词**：定期优化AI提示词，提高报告质量
