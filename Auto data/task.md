# Athena智能数据查询系统 - 开发任务计划

## 项目概述
本项目旨在构建一个基于AWS Athena的智能数据查询系统，解决以下核心场景问题：
1. 为非技术用户提供便捷的数据查询方案，使其无需掌握SQL技能即可通过数据库快速查询数据
2. 处理大规模数据分包导出excel，解决传统Excel数据量限制问题
3. 满足多样化查询需求，提升查询效率，避免纯人工处理带来的低效问题

## 项目实施阶段

### 第一阶段：后台系统开发 (当前阶段)
- 支持用户输入自然语言文本，自动转换为标准SQL语句
- 通过接口执行数据库查询，并将结果返回至后台页面展示
- 提供数据查看及大规模数据分包导出功能

### 第二阶段：自动化查询功能
- 实现定时查询任务配置功能
- 自动执行预定查询并生成结果报告
- 支持通过电子邮件/TG等渠道发送查询报告

### 第三阶段：智能客服集成
- 集成TG机器人/飞书应用
- 通过机器人/应用处理用户的数据查询请求
- 自动将查询结果发送至指定TG群组/飞书群组

**优先级**: 高

## 核心用户场景

### 场景1：业务分析师数据查询
**用户角色**：业务分析师
**使用场景**：需要快速获取销售数据进行分析
**输入示例**："我需要查看2024年1月到3月各个产品线的销售额和订单量"
**系统处理**：
1. 解析时间范围（2024年1-3月）和查询维度（产品线、销售额、订单量）
2. 生成Athena SQL查询语句
3. 检测数据量，如超过10万条自动按月份拆分
4. 执行查询并导出Excel文件

### 场景2：运营人员用户行为分析
**用户角色**：运营人员
**使用场景**：分析用户活跃度和行为路径
**输入示例**："帮我导出上周新注册用户的行为数据，包括页面访问和购买转化"
**系统处理**：
1. 识别时间范围（上周）和用户群体（新注册用户）
2. 关联多张表生成复杂查询
3. 自动拆分大数据集避免单文件过大
4. 生成多个Excel文件并打包下载

### 场景3：财务人员报表生成
**用户角色**：财务人员
**使用场景**：生成月度财务报表
**输入示例**："生成12月份的收入成本报表，按部门和项目分组"
**系统处理**：
1. 解析财务指标和分组维度
2. 生成符合财务规范的SQL查询
3. 处理大数据集并按部门拆分文件
4. 导出格式化的Excel报表

name: "游戏回合记录查询"
description: "查询和分析指定回合的游戏记录异常情况"
parameters:
  - name: "MerchantID"
    type: "string"
    required: true
    validation: "^[0-9]+$"
    aliases: ["商户ID", "merchant_id", "merchantid"]
  - name: "biz_id"
    type: "string"
    required: true
    validation: "^[a-zA-Z0-9_-]+$"

  - name: "round_id"
    type: "string"
    required: true
    validation: "^[0-9]+$"
    aliases: ["回合ID", "roundid", "round"]
query_template: |
  SELECT 
    r.*,
    u.username,
    g.game_name,
    t.transaction_amount,
    t.transaction_status
  FROM game_rounds r
  LEFT JOIN users u ON r.user_id = u.id
  LEFT JOIN games g ON r.game_id = g.id
  LEFT JOIN transactions t ON r.round_id = t.round_id
  WHERE r.merchant_id = '${MerchantID}'
  AND r.biz_id = '${biz_id}'
  AND r.round_id = '${round_id}'
anomaly_rules:
  - name: "金额异常检测"
    condition: "bet_amount > 10000 OR win_amount > 50000"
    level: "high"
    message: "投注或中奖金额异常偏高"
  - name: "时间异常检测"
    condition: "TIMESTAMPDIFF(SECOND, start_time, end_time) < 1"
    level: "medium"
    message: "游戏时长异常偏短"
  - name: "状态异常检测"
    condition: "game_status NOT IN ('completed', 'settled')"
    level: "high"
    message: "游戏状态异常"
export_rules:
  format: "json"
  include_analysis: true
  fields:
    - {name: "回合ID", column: "round_id"}
    - {name: "用户名", column: "username"}
    - {name: "游戏名称", column: "game_name"}

```yaml
name: "时间范围数据查询"
description: "查询指定时间范围内的货币数据"
parameters:
  - name: "start_date"
    type: "date"
    required: true
    aliases: ["开始时间", "从", "号"]
  - name: "end_date"
    type: "date"
    required: true
    aliases: ["结束时间", "到", "号"]
  - name: "currencies"
    type: "array"
    required: true
    values: ["PHP", "INR", "USD", "CNY", "THB"]
    aliases: ["货币", "币种"]
text_patterns:
  - pattern: "(\d+)号到(\d+)号的([A-Z]{3})跟([A-Z]{3})数据"
    mapping:
      start_date: "2024-01-{1}"
      end_date: "2024-01-{2}"
      currencies: ["{3}", "{4}"]
query_template: |
  SELECT 
    DATE(created_at) as date,
    currency,
    SUM(amount) as total_amount,
    COUNT(*) as transaction_count,
    AVG(exchange_rate) as avg_exchange_rate
  FROM transactions 
  WHERE DATE(created_at) BETWEEN '${start_date}' AND '${end_date}'
  AND currency IN (${currencies})
  GROUP BY DATE(created_at), currency
  ORDER BY date, currency
export_rules:
  format: "excel"
  filename: "currency_data_${start_date}_to_${end_date}.xlsx"
  sheets:
    - name: "数据汇总"
      charts: ["时间序列图", "货币对比图"]

name: "业务指标报表生成"
description: "生成包含多个业务指标的定期报表"
parameters:
  - name: "metrics"
    type: "array"
    required: true
    values: ["MAU", "DAU", "retention", "net_revenue", "ARPU", "LTV"]
    aliases: ["月活", "日活", "留存", "净营收", "指标"]
  - name: "report_period"
    type: "enum"
    values: ["daily", "weekly", "monthly"]
    required: true
    aliases: ["日报", "周报", "月报"]
  - name: "auto_send"
    type: "boolean"
    default: false
    aliases: ["自动发送", "定期发送"]
text_patterns:
  - pattern: "(月活|日活|留存|净营收).*?(月报|周报|日报).*?发送"
    mapping:
      metrics: ["MAU", "DAU", "retention", "net_revenue"]
      report_period: "monthly"
      auto_send: true
query_templates:
  MAU: |
    SELECT COUNT(DISTINCT user_id) as mau
    FROM user_activities 
    WHERE DATE(created_at) >= DATE_SUB(CURDATE(), INTERVAL 30 DAY)
  DAU: |
    SELECT DATE(created_at) as date, COUNT(DISTINCT user_id) as dau
    FROM user_activities 
    WHERE DATE(created_at) >= DATE_SUB(CURDATE(), INTERVAL 7 DAY)
    GROUP BY DATE(created_at)
  retention: |
    SELECT 
      cohort_month,
      period_number,
      COUNT(DISTINCT user_id) / first_value(COUNT(DISTINCT user_id)) OVER (PARTITION BY cohort_month ORDER BY period_number) as retention_rate
    FROM user_cohort_analysis
    GROUP BY cohort_month, period_number
  net_revenue: |
    SELECT 
      DATE(created_at) as date,
      SUM(amount - cost) as net_revenue
    FROM revenue_records 
    WHERE DATE(created_at) >= DATE_SUB(CURDATE(), INTERVAL 30 DAY)
    GROUP BY DATE(created_at)
export_rules:
  format: "pdf"
  template: "business_metrics_template.html"
  filename: "${report_period}_metrics_report_${date}.pdf"
  auto_schedule:
    enabled: "${auto_send}"
    frequency: "${report_period}"
    recipients: ["analytics@company.com", "management@company.com"]
```
