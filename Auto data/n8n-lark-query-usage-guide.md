# Lark消息查询处理器 - 完整使用指南

## 🎯 功能概述

这个查询处理器支持**双向查询**：
- **正向查询**：商户名 → 商户ID
- **反向查询**：商户ID → 商户名

## 📋 支持的查询格式

### 1. 正向查询（商户名查ID）

| 输入格式 | 示例 | 输出 |
|---------|------|------|
| 直接商户名 | `betfiery` | 找到商户: betfiery (ID: 1698202251) |
| 中文格式 | `商户betfiery的id` | 找到商户: betfiery (ID: 1698202251) |
| 带查询前缀 | `查询betfiery` | 找到商户: betfiery (ID: 1698202251) |
| 带后缀 | `betfiery的信息` | 找到商户: betfiery (ID: 1698202251) |
| 模糊匹配 | `bet` | 找到相似商户: betfiery (ID: 1698202251) |

### 2. 反向查询（ID查商户名）

| 输入格式 | 示例 | 输出 |
|---------|------|------|
| 纯数字ID | `1698202251` | 通过ID找到商户: betfiery (ID: 1698202251) |
| 中文格式 | `商户1698202251的名称` | 通过ID找到商户: betfiery (ID: 1698202251) |

## 🔧 返回结果格式

### 成功查询结果
```javascript
{
  "queryType": "merchant_exact",        // 查询类型
  "queryText": "betfiery",             // 原始查询文本
  "extractedQuery": "betfiery",        // 提取的关键词
  "result": {
    "success": true,
    "merchant_id": 1698202251,
    "sub_merchant_name": "betfiery",
    "main_merchant_name": "RD1",
    "message": "找到商户: betfiery (ID: 1698202251)"
  }
}
```

### 查询类型说明

| 查询类型 | 说明 | 示例 |
|---------|------|------|
| `merchant_exact` | 精确匹配商户名 | betfiery → betfiery |
| `merchant_by_id` | 通过ID找到商户 | 1698202251 → betfiery |
| `merchant_fuzzy` | 模糊匹配商户名 | bet → betfiery |
| `no_match` | 未找到匹配 | xyz → 未找到 |
| `id_not_found` | ID不存在 | 9999999 → ID不存在 |
| `no_data` | 没有商户数据 | - |

## 🔍 故障排除

### 问题1：返回"没有可用的商户数据"
**解决方案：**
1. 检查上游节点是否正确传递了商户数据
2. 查看控制台日志确认数据结构
3. 确保商户数据包含 `merchant_id` 和 `sub_merchant_name` 字段

### 问题2：ID查询不工作
**解决方案：**
1. 确保输入的是纯数字（不包含空格或其他字符）
2. 检查商户数据中的ID字段类型（数字 vs 字符串）
3. 查看控制台日志确认ID匹配过程

### 问题3：中文查询不工作
**解决方案：**
1. 确保查询格式正确："商户betfiery的id"
2. 检查是否有额外的空格或特殊字符
3. 尝试直接输入英文商户名
