# 如何启动服务

## 快速启动指南

### 方法一：使用启动脚本（推荐）

#### 1. 启动PDF服务
在项目根目录打开命令行，运行：
```cmd
start-pdf-service.bat
```

或者手动启动：
```cmd
cd backend\pdf-service
node server.js
```

**预期输出**：
```
HTML->PDF service listening on http://0.0.0.0:8787
Found Chrome at: C:\Program Files\Google\Chrome\Application\chrome.exe
```

#### 2. 启动主后端服务
在项目根目录打开**新的**命令行窗口，运行：
```cmd
start-server.bat
```

或者手动启动：
```cmd
cd backend
node server.js
```

**预期输出**：
```
Server is running on port 8000
```

### 方法二：手动启动（分步骤）

#### 步骤1：启动PDF服务
```cmd
# 进入PDF服务目录
cd backend\pdf-service

# 安装依赖（首次运行）
npm install

# 启动服务
node server.js
```

#### 步骤2：启动主后端服务
**打开新的命令行窗口**，运行：
```cmd
# 进入后端目录
cd backend

# 安装依赖（首次运行）
npm install

# 启动服务
node server.js
```

## 验证服务是否启动成功

### 1. 检查端口占用
```cmd
netstat -ano | findstr "8787 8000"
```

应该看到：
```
TCP    0.0.0.0:8000           0.0.0.0:0              LISTENING       <PID>
TCP    0.0.0.0:8787           0.0.0.0:0              LISTENING       <PID>
```

### 2. 测试PDF服务
```cmd
curl http://localhost:8787/health
```

应该返回：
```json
{"status":"ok","timestamp":"2026-01-14T..."}
```

### 3. 测试主后端服务
在浏览器访问：
```
http://localhost:8000
```

## 常见问题

### 问题1：端口已被占用
**错误信息**：`Error: listen EADDRINUSE: address already in use`

**解决方案**：
1. 查找占用端口的进程：
   ```cmd
   netstat -ano | findstr "8787"
   ```

2. 终止进程：
   ```cmd
   taskkill /F /PID <进程ID>
   ```

3. 重新启动服务

### 问题2：Chrome浏览器找不到
**错误信息**：`Chrome executable not found`

**解决方案**：
1. 安装Google Chrome浏览器：https://www.google.com/chrome/
2. 或设置环境变量：
   ```cmd
   set CHROME_PATH=C:\你的Chrome路径\chrome.exe
   ```

### 问题3：依赖未安装
**错误信息**：`Cannot find module 'express'`

**解决方案**：
```cmd
# 在backend目录
cd backend
npm install

# 在pdf-service目录
cd backend\pdf-service
npm install
```

## 停止服务

### 方法1：在命令行窗口按 Ctrl+C

### 方法2：强制终止进程
```cmd
# 查找进程
netstat -ano | findstr "8787 8000"

# 终止进程
taskkill /F /PID <进程ID>
```

## 重启服务

如果修改了代码，需要重启服务才能生效：

1. 停止所有服务（Ctrl+C 或 taskkill）
2. 重新运行启动脚本

## 启动顺序

**重要**：必须按以下顺序启动：
1. 先启动 PDF服务 (端口8787)
2. 再启动 主后端服务 (端口8000)

两个服务都需要保持运行状态。

## 开发模式

如果需要自动重启（代码修改后自动重启），可以使用nodemon：

```cmd
# 安装nodemon（全局）
npm install -g nodemon

# 使用nodemon启动
cd backend
nodemon server.js

# PDF服务
cd backend\pdf-service
nodemon server.js
```

## 生产环境部署

生产环境建议使用PM2进程管理器：

```cmd
# 安装PM2
npm install -g pm2

# 启动服务
pm2 start backend/server.js --name "main-backend"
pm2 start backend/pdf-service/server.js --name "pdf-service"

# 查看状态
pm2 status

# 查看日志
pm2 logs

# 停止服务
pm2 stop all

# 重启服务
pm2 restart all
```
