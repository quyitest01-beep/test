# ⚠️ 重要：数据源说明

## 问题诊断

如果 N8N 显示 **"No output data returned"**，说明没有找到留存数据。

## 📊 数据文件说明

你的项目中有三个不同的数据文件：

### 1. **shangyou.json** - 营收数据 ❌ 不是留存数据
```json
{
  "日期": "2025-12-21",
  "商户名": "sortebot",
  "货币": "ARS",
  "游戏": "Scratch Match",
  "总投注": "950.00",
  "RTP": "47.37%",
  "GGR": "500.00"
}
```
**用途**: 包含游戏营收、投注、派奖等数据  
**不包含**: 留存数据（d0_users, d1_retention_rate 等）

### 2. **shangy.json** - 商户映射数据 ✅ 需要
```json
{
  "status": "success",
  "filtered_merchants": [
    {
      "sub_merchant_name": "betfiery",
      "merchant_id": 1698202251
    }
  ]
}
```
**用途**: 提供 merchant_id 到商户名的映射  
**必需**: 用于匹配商户名

### 3. **xiayou.json** - 留存数据 ✅ 必需
```json
{
  "target_game": {
    "english_name": "Keno"
  },
  "metrics": {
    "global": {
      "retention_new": [
        {
          "merchant_id": 1698202251,
          "d0_users": 1000,
          "d1_users": 850,
          "d1_retention_rate": "85.0"
        }
      ]
    }
  }
}
```
**用途**: 包含留存数据（d0_users, d1_retention_rate 等）  
**必需**: 这是主要的数据源

## ✅ 正确的 N8N 配置

### 方案 1: 使用两个数据源（推荐）

```
[读取 shangy.json] ─┐
                    ├─→ [Merge] ─→ [Code节点] ─→ 输出
[读取 xiayou.json] ─┘
```

### 方案 2: 使用一个数据源（如果已合并）

```
[读取合并后的数据] ─→ [Code节点] ─→ 输出
```

## ❌ 错误的配置

```
[读取 shangyou.json] ─→ [Code节点] ─→ ❌ No output
```
**问题**: shangyou.json 只包含营收数据，没有留存数据

## 🔍 如何检查你的数据

### 检查是否有留存数据字段：

**留存数据必须包含以下字段之一：**
- `d0_users` (当日用户数)
- `d1_users` (次日用户数)
- `d1_retention_rate` (次日留存率)
- `d7_users` (7日用户数)
- `d7_retention_rate` (7日留存率)

**或者包含以下结构：**
- `metrics.global.retention_new` (新用户留存数组)
- `metrics.global.retention_active` (活跃用户留存数组)

### 检查你的输入数据：

在 N8N 的 INPUT 面板中查看：
1. 是否有 `retention_new` 或 `retention_active` 字段？
2. 是否有 `d0_users`, `d1_retention_rate` 等字段？
3. 如果只看到 `总投注`, `RTP`, `GGR` 等字段，说明这是营收数据，不是留存数据

## 🚀 解决方案

### 步骤 1: 确认数据源
检查你的数据源是否包含 **xiayou.json** 格式的留存数据

### 步骤 2: 添加正确的数据源
在 N8N 中添加读取 **xiayou.json** 的节点

### 步骤 3: 合并数据（可选）
如果需要商户名映射，同时添加 **shangy.json** 数据源，并使用 Merge 节点合并

### 步骤 4: 连接 Code 节点
将合并后的数据连接到 Code 节点

## 💡 常见问题

### Q: 我只有 shangyou.json，能处理吗？
**A**: 不能。shangyou.json 是营收数据，不包含留存数据。你需要 xiayou.json。

### Q: shangy.json 和 shangyou.json 有什么区别？
**A**: 
- **shangy.json**: 商户映射数据（merchant_id → 商户名）
- **shangyou.json**: 营收数据（投注、派奖、RTP、GGR）

### Q: 我必须同时使用 shangy.json 和 xiayou.json 吗？
**A**: 
- **xiayou.json**: 必需（包含留存数据）
- **shangy.json**: 可选（用于更准确的商户名匹配）

如果只使用 xiayou.json，代码会使用 xiayou.json 中的原始商户名。
