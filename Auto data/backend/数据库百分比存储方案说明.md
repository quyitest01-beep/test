# 数据库百分比存储方案说明

## 问题

数据库可以存百分比的数值吗？格式是什么，比如"80.22%"？

## 答案

**数据库不建议存储字符串格式的百分比（如 "80.22%"），应该存储为数值类型，在显示时再格式化为百分比字符串。**

## 存储方案对比

### 方案1：存储为百分比数值（推荐）✅

**设计**：使用 `DECIMAL` 类型存储百分比数值（0-100）

**格式**：
- 存储值：`80.22`（数值）
- 显示值：`"80.22%"`（在应用层格式化）

**表结构**：
```sql
CREATE TABLE example (
  `retention_rate` DECIMAL(10, 4) DEFAULT 0.0000 COMMENT '留存率（%，0-100）',
  `rtp` DECIMAL(10, 4) DEFAULT 0.0000 COMMENT 'RTP（%，0-100）'
);
```

**数据示例**：
```sql
-- 存储 80.22%
INSERT INTO example (retention_rate) VALUES (80.22);

-- 查询并格式化显示
SELECT 
  retention_rate,
  CONCAT(retention_rate, '%') as retention_rate_formatted
FROM example;
-- 结果：retention_rate = 80.2200, retention_rate_formatted = "80.22%"
```

**优点**：
- ✅ 可以进行数值计算和比较
- ✅ 支持排序、聚合函数（SUM、AVG等）
- ✅ 存储空间小
- ✅ 查询性能好

**缺点**：
- ⚠️ 需要在应用层或查询时格式化为 "80.22%"

---

### 方案2：存储为小数（0-1）

**设计**：使用 `DECIMAL` 类型存储小数（0-1）

**格式**：
- 存储值：`0.8022`（小数）
- 显示值：`"80.22%"`（在应用层格式化：值 * 100 + "%"）

**表结构**：
```sql
CREATE TABLE example (
  `retention_rate` DECIMAL(10, 4) DEFAULT 0.0000 COMMENT '留存率（0-1）',
  `rtp` DECIMAL(5, 4) DEFAULT 0.0000 COMMENT 'RTP（0-1）'
);
```

**数据示例**：
```sql
-- 存储 80.22%（即 0.8022）
INSERT INTO example (retention_rate) VALUES (0.8022);

-- 查询并格式化显示
SELECT 
  retention_rate,
  CONCAT(ROUND(retention_rate * 100, 2), '%') as retention_rate_formatted
FROM example;
-- 结果：retention_rate = 0.8022, retention_rate_formatted = "80.22%"
```

**优点**：
- ✅ 符合数学计算习惯（0-1范围）
- ✅ 可以直接用于计算

**缺点**：
- ⚠️ 显示时需要乘以100
- ⚠️ 不够直观（0.8022 vs 80.22）

---

### 方案3：存储为字符串（不推荐）❌

**设计**：使用 `VARCHAR` 类型存储 "80.22%"

**格式**：
- 存储值：`"80.22%"`（字符串）
- 显示值：`"80.22%"`（直接显示）

**表结构**：
```sql
CREATE TABLE example (
  `retention_rate` VARCHAR(20) DEFAULT '0%' COMMENT '留存率（字符串格式）'
);
```

**数据示例**：
```sql
-- 存储 "80.22%"
INSERT INTO example (retention_rate) VALUES ('80.22%');

-- 查询
SELECT retention_rate FROM example;
-- 结果：retention_rate = "80.22%"
```

**优点**：
- ✅ 显示时不需要格式化

**缺点**：
- ❌ **无法进行数值计算**（无法 SUM、AVG、比较大小）
- ❌ **无法排序**（字符串排序：'100%' < '20%'）
- ❌ **存储空间大**
- ❌ **查询性能差**
- ❌ **数据验证困难**

**示例问题**：
```sql
-- ❌ 无法正确计算平均值
SELECT AVG(retention_rate) FROM example;  -- 错误！

-- ❌ 无法正确排序
SELECT * FROM example ORDER BY retention_rate;
-- 结果：'100%' 会排在 '20%' 前面（字符串排序）

-- ❌ 无法比较大小
SELECT * FROM example WHERE retention_rate > 50;  -- 错误！
```

## 推荐方案：方案1（存储为百分比数值）

### 当前表结构设计

在我们的表结构中，已经使用了方案1：

```sql
-- RTP字段
`rtp` DECIMAL(10, 4) GENERATED ALWAYS AS (
  CASE 
    WHEN `total_bet` > 0 THEN (`total_payout` / `total_bet`) * 100 
    ELSE 0 
  END
) STORED COMMENT 'RTP（%，0-100）'

-- 留存率字段
`d1_retention_rate` DECIMAL(10, 4) GENERATED ALWAYS AS (
  CASE 
    WHEN `daily_users` > 0 THEN (`d1_users` / `daily_users`) * 100 
    ELSE 0 
  END
) STORED COMMENT 'D1留存率（%，0-100）'
```

### 数据插入示例

```sql
-- 直接插入百分比数值
INSERT INTO game_retention_data (daily_users, d1_users, ...)
VALUES (191, 40, ...);
-- d1_retention_rate 会自动计算为 20.9424（表示 20.9424%）

-- 如果需要手动插入百分比值
UPDATE game_retention_data 
SET d1_retention_rate = 80.22 
WHERE id = 1;
-- 存储值：80.22（表示 80.22%）
```

### 查询和显示

```sql
-- 方式1：直接查询数值
SELECT d1_retention_rate FROM game_retention_data;
-- 结果：20.9424

-- 方式2：查询时格式化
SELECT 
  d1_retention_rate,
  CONCAT(ROUND(d1_retention_rate, 2), '%') as d1_retention_rate_formatted
FROM game_retention_data;
-- 结果：d1_retention_rate = 20.9424, d1_retention_rate_formatted = "20.94%"

-- 方式3：使用 FORMAT 函数（MySQL）
SELECT 
  d1_retention_rate,
  CONCAT(FORMAT(d1_retention_rate, 2), '%') as d1_retention_rate_formatted
FROM game_retention_data;
-- 结果：d1_retention_rate = 20.9424, d1_retention_rate_formatted = "20.94%"
```

### 应用层格式化（JavaScript）

```javascript
// 格式化百分比
function formatPercentage(value, decimals = 2) {
  if (value === null || value === undefined) return '0%';
  return `${Number(value).toFixed(decimals)}%`;
}

// 使用示例
const retentionRate = 20.9424;
console.log(formatPercentage(retentionRate)); // "20.94%"
console.log(formatPercentage(retentionRate, 1)); // "20.9%"
```

### 数据类型选择

**DECIMAL(10, 4)**：
- 总长度：10位
- 小数位：4位
- 范围：-999999.9999 到 999999.9999
- 对于百分比（0-100）：足够使用
- 精度：保留4位小数，如 80.2222%

**DECIMAL(5, 2)**：
- 总长度：5位
- 小数位：2位
- 范围：-999.99 到 999.99
- 对于百分比（0-100）：足够使用
- 精度：保留2位小数，如 80.22%

**建议**：
- 如果只需要2位小数精度：使用 `DECIMAL(5, 2)`
- 如果需要更高精度：使用 `DECIMAL(10, 4)`

## 总结

1. **推荐方案**：存储为百分比数值（DECIMAL类型，0-100范围）
   - 存储：`80.22`（数值）
   - 显示：`"80.22%"`（在应用层格式化）

2. **数据类型**：
   - `DECIMAL(5, 2)` - 2位小数精度（推荐用于百分比）
   - `DECIMAL(10, 4)` - 4位小数精度（更高精度需求）

3. **格式化方式**：
   - SQL：`CONCAT(ROUND(value, 2), '%')`
   - JavaScript：`${value.toFixed(2)}%`
   - Python：`f"{value:.2f}%"`

4. **避免**：不要存储字符串格式的百分比（如 "80.22%"）
