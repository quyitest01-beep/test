# n8n 工作流配置 - 过滤数据1中不在数据2中的数据

## 需求

输出数据1中没有出现在数据2中的数据（基于 `game_id` 匹配）。

**数据1结构**：
```json
{
  "game_id": 1698217745002,
  "game_name": "Fortune Mouse"
}
```

**数据2结构**：
```json
{
  "id": 1,
  "createdAt": "2025-11-07T03:48:13.422Z",
  "updatedAt": "2025-11-07T03:48:13.422Z",
  "game_id": "1698217736002",
  "game": "Lucky Tanks",
  "merchant": "1698203185"
}
```

**注意**：
- 数据1的 `game_id` 是**数字**
- 数据2的 `game_id` 是**字符串**
- 匹配时需要统一转换为字符串

## 方案1：使用 Merge 节点 + Code 节点（推荐）

### 工作流结构

```
Input1 (数据1) ──┐
                 ├──→ Merge Node ──→ Code Node (过滤) ──→ Output
Input2 (数据2) ──┘
```

### 步骤1：Merge 节点配置

**配置**：
```json
{
  "parameters": {
    "mode": "combine",
    "fieldsToMatchString": "",
    "joinMode": "keepEverything",
    "options": {}
  }
}
```

**说明**：
- `mode: "combine"` - 简单合并两个输入
- `joinMode: "keepEverything"` - 保留所有数据
- 目的：将两个数据源合并后传入 Code 节点

### 步骤2：Code 节点配置

使用 `backend/filter-game-data-standalone.js` 代码。

**功能**：
1. 自动识别数据1和数据2（根据字段特征）
2. 构建数据2的 `game_id` 集合
3. 过滤数据1，只保留不在数据2中的数据
4. 处理 `game_id` 类型差异（数字 vs 字符串）

## 方案2：使用两个 Code 节点 + 变量传递

### 工作流结构

```
Input2 (数据2) ──→ Code Node A (收集 game_id) ──→ Set Variable
                                                       │
Input1 (数据1) ──→ Code Node B (读取变量，过滤) ──→ Output
```

### 步骤1：Code Node A（收集数据2的 game_id）

```javascript
const inputs = $input.all();
const gameIds = new Set();

inputs.forEach((inputItem) => {
  const item = inputItem.json;
  if (item && item.game_id !== undefined && item.game_id !== null) {
    gameIds.add(String(item.game_id).trim());
  }
});

// 输出 game_id 列表（传递给下一个节点）
return [{
  json: {
    gameIds: Array.from(gameIds),
    count: gameIds.size
  }
}];
```

### 步骤2：Code Node B（过滤数据1）

```javascript
const inputs = $input.all();
// 从上一个节点获取 gameIds
// 注意：需要确保数据流正确连接

// 假设通过某种方式获取 data2GameIds
// 这里需要根据实际工作流调整

const data2GameIds = new Set(/* 从上游或变量获取 */);

const results = [];

inputs.forEach((inputItem) => {
  const item = inputItem.json;
  if (item && item.game_id !== undefined) {
    const gameIdStr = String(item.game_id).trim();
    if (!data2GameIds.has(gameIdStr)) {
      results.push(inputItem);
    }
  }
});

return results;
```

## 方案3：直接使用 Code 节点（最简单）

### 工作流结构

```
Input1 (数据1) ──┐
                 ├──→ Merge Node (combine) ──→ Code Node ──→ Output
Input2 (数据2) ──┘
```

### Merge 节点配置

```json
{
  "parameters": {
    "mode": "combine",
    "options": {}
  }
}
```

### Code 节点

直接使用 `backend/filter-game-data-standalone.js` 代码。

**优点**：
- 简单直接
- 自动识别两个数据源
- 自动处理类型转换
- 详细的日志输出

## 数据识别逻辑

代码会自动识别两个数据源：

**数据1特征**：
- 有 `game_name` 字段
- 没有 `createdAt` 字段
- 没有 `merchant` 字段
- `game_id` 是数字类型

**数据2特征**：
- 有 `createdAt` 字段
- 或有 `merchant` 字段
- 或有 `game` 字段（但不是 `game_name`）
- `game_id` 是字符串类型

## 匹配逻辑

1. **类型转换**：将所有 `game_id` 转换为字符串进行比较
2. **去重**：数据2的 `game_id` 使用 Set 去重
3. **过滤**：数据1中 `game_id` 不在数据2集合中的数据被保留

## 输出格式

输出过滤后的数据1，格式与输入相同：

```json
{
  "game_id": 1698217745002,
  "game_name": "Fortune Mouse"
}
```

## 调试建议

1. **检查数据识别**：
   - 查看日志中的"数据1"和"数据2"数量
   - 确认识别是否正确

2. **检查匹配**：
   - 查看"匹配的数据"和"未匹配的数据"数量
   - 查看具体的匹配/跳过日志

3. **检查 game_id 类型**：
   - 确认数据1的 `game_id` 是数字
   - 确认数据2的 `game_id` 是字符串
   - 代码会自动处理类型转换

4. **测试数据**：
   - 使用少量测试数据验证
   - 确认过滤逻辑正确

## 常见问题

### Q1: 所有数据都被过滤掉了？

**可能原因**：
- 数据识别错误
- `game_id` 类型不匹配
- 数据2包含了所有数据1的 `game_id`

**解决方法**：
- 查看日志，确认数据识别是否正确
- 检查 `game_id` 的值和类型
- 验证匹配逻辑

### Q2: 没有数据被过滤？

**可能原因**：
- 数据2为空
- 数据识别错误（两个数据源都识别为数据1）

**解决方法**：
- 查看日志中的"数据2"数量
- 检查数据2的字段是否符合预期

### Q3: Merge 节点配置问题？

**解决方法**：
- 使用 `mode: "combine"` 简单合并
- 然后在 Code 节点中处理过滤逻辑
- 这是最可靠的方法

## 推荐配置

**最简单可靠的方法**：

1. Merge 节点：`mode: "combine"`
2. Code 节点：使用 `backend/filter-game-data-standalone.js`

这样就不需要复杂的 Merge 节点配置，所有逻辑都在 Code 节点中处理。
