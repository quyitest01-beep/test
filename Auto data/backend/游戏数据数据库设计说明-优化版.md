# 游戏数据数据库表结构设计说明（优化版）

## 一、设计目标

- **数据用途**：生成数据报告、日常回复客户数据问题
- **数据范围**：仅存储GMP/popular厂商游戏数据
- **数据分类**：营收数据、用户数据、留存数据

## 二、表结构概览

### 核心数据表（5个）

1. **revenue_data** - 营收数据表
2. **game_user_data** - 游戏用户数据表
3. **merchant_user_data** - 商户用户数据表
4. **game_retention_data** - 游戏留存数据表
5. **merchant_retention_data** - 商户留存数据表


## 三、表结构详细说明

### 1. revenue_data（营收数据表）

**用途**：存储按商户、游戏、币种的营收数据

**字段说明**：
- `date`: 日期（DATE类型）
- `merchant_name`: 商户名
- `game_name`: 游戏名
- `currency`: 币种（USD, MXN, BRL等）
- `total_bet`: 总投注（USD，DECIMAL(20,4)）
- `total_payout`: 总派奖（USD，DECIMAL(20,4)）
- `total_rounds`: 总局数（BIGINT）
- `total_ggr`: 总GGR（计算字段：total_bet - total_payout）
- `rtp`: RTP（计算字段：(total_payout / total_bet) * 100）
- `provider`: 厂商（GMP/popular）

**索引**：
- 单列索引：`date`、`merchant_name`、`game_name`、`currency`、`provider`
- 复合索引：`idx_date_merchant`、`idx_date_game`、`idx_date_currency`
- 唯一索引：`uk_date_merchant_game_currency`（防止重复数据）

**数据示例**：
```sql
INSERT INTO revenue_data (date, merchant_name, game_name, currency, total_bet, total_payout, total_rounds, provider)
VALUES ('2025-10-31', 'Betfarms', 'Bank Heist', 'USD', 121038.3167, 114553.5091, 575922, 'GMP');
```

### 2. game_user_data（游戏用户数据表）

**用途**：存储按游戏、日期的投注用户数

**字段说明**：
- `date`: 日期（DATE类型，NULL表示汇总数据）
- `period_range`: 周期范围（汇总数据使用，格式：YYYYMMDD-YYYYMMDD 或 YYYYMM）
- `data_type`: 数据类型（daily=日数据，weekly=周汇总，monthly=月汇总）
- `game_name`: 游戏名
- `bet_users`: 投注用户数（INT UNSIGNED）
- `provider`: 厂商（GMP/popular）

**数据存储规则**：
- **日数据**：`date IS NOT NULL`，`period_range IS NULL`，`data_type = 'daily'`
- **周汇总数据（合计）**：`date IS NULL`，`period_range = '20251027-20251102'`，`data_type = 'weekly'`
- **月汇总数据（合计）**：`date IS NULL`，`period_range = '202511'`，`data_type = 'monthly'`

**索引**：
- 单列索引：`date`、`period_range`、`data_type`、`game_name`、`provider`
- 复合索引：`idx_date_game`、`idx_period_game`
- 唯一索引：
  - `uk_date_game`：日数据唯一性（date + game_name + data_type）
  - `uk_period_game_type`：汇总数据唯一性（period_range + game_name + data_type）

**数据示例**：
```sql
-- 日数据
INSERT INTO game_user_data (date, game_name, bet_users, data_type, provider)
VALUES ('2025-10-31', 'Bank Heist', 1726, 'daily', 'GMP');

-- 周汇总数据（合计）
INSERT INTO game_user_data (date, period_range, game_name, bet_users, data_type, provider)
VALUES (NULL, '20251027-20251102', 'Bank Heist', 1726, 'weekly', 'GMP');

-- 月汇总数据（合计）
INSERT INTO game_user_data (date, period_range, game_name, bet_users, data_type, provider)
VALUES (NULL, '202511', 'Bank Heist', 5000, 'monthly', 'GMP');
```

### 4. game_retention_data（游戏留存数据表）

**用途**：存储按游戏、日期的留存数据

**字段说明**：
- `date`: 日期（DATE类型）
- `game_name`: 游戏名

- `d3_users`: 3日用户数（D3留存）

- `d14_users`: 14日用户数（D14留存）
- `d30_users`: 30日用户数（D30留存）
- `d1_retention_rate`: D1留存率（计算字段，%）

- `provider`: 厂商（GMP/popular）

**索引**：
- 单列索引：`date`、`game_name`、`merchant_name`、`user_type`、`provider`、`daily_users`
- 复合索引：`idx_date_game`、`idx_date_game_type`
- 唯一索引：`uk_date_game_merchant_type`（防止重复数据）

**数据示例**：

## 五、设计特点

### 1. 计算字段
- `total_ggr`: 自动计算（total_bet - total_payout）
- `rtp`: 自动计算（(total_payout / total_bet) * 100）
- `d1_retention_rate`: 自动计算（(d1_users / daily_users) * 100）
- `d7_retention_rate`: 自动计算（(d7_users / daily_users) * 100）

### 2. 唯一索引
- 防止重复数据插入
- 支持 `ON DUPLICATE KEY UPDATE` 进行数据更新

### 3. 索引优化
- 单列索引：支持单条件查询
- 复合索引：优化多条件组合查询
- 覆盖常用查询场景

### 4. 数据类型
- `DATE`: 使用DATE类型存储日期，便于日期范围查询
- `DECIMAL(20,4)`: 精确存储金额数据
- `INT UNSIGNED`: 存储用户数（非负数）

## 六、常用查询场景

### 1. 生成数据报告

#### 查询某日期范围的营收汇总
```sql
SELECT 
  date,
  merchant_name,
  game_name,
  currency,
  SUM(total_bet) as total_bet,
  SUM(total_payout) as total_payout,
  SUM(total_ggr) as total_ggr,
  SUM(total_rounds) as total_rounds
FROM revenue_data
WHERE date BETWEEN '2025-10-27' AND '2025-11-02'
  AND provider IN ('GMP', 'popular')
GROUP BY date, merchant_name, game_name, currency
ORDER BY date, total_ggr DESC;
```

#### 查询游戏用户数据汇总（日数据）
```sql
SELECT 
  date,
  game_name,
  bet_users
FROM game_user_data
WHERE date BETWEEN '2025-10-27' AND '2025-11-02'
  AND data_type = 'daily'
  AND provider IN ('GMP', 'popular')
ORDER BY date, bet_users DESC;
```

#### 查询游戏用户数据汇总（周汇总数据）
```sql
SELECT 
  period_range,
  game_name,
  bet_users
FROM game_user_data
WHERE period_range = '20251027-20251102'
  AND data_type = 'weekly'
  AND provider IN ('GMP', 'popular')
ORDER BY bet_users DESC;
```

#### 查询留存数据Top 20
```sql
SELECT 
  game_name,
  merchant_name,
  daily_users,
  d1_users,
  d7_users,
  d1_retention_rate,
  d7_retention_rate
FROM game_retention_data
WHERE date = '2025-10-31'
  AND user_type = 'new_user'
  AND daily_users >= 50
  AND provider IN ('GMP', 'popular')
ORDER BY d1_retention_rate DESC
LIMIT 20;
```

### 2. 回复客户数据问题

#### 查询某个游戏的详细数据

## 七、数据插入建议

### 1. 使用 ON DUPLICATE KEY UPDATE
```sql
INSERT INTO revenue_data (date, merchant_name, game_name, currency, total_bet, total_payout, total_rounds, provider)
VALUES ('2025-10-31', 'Betfarms', 'Bank Heist', 'USD', 121038.3167, 114553.5091, 575922, 'GMP')
ON DUPLICATE KEY UPDATE 
  total_bet = VALUES(total_bet),
  total_payout = VALUES(total_payout),
  total_rounds = VALUES(total_rounds),
  updated_at = CURRENT_TIMESTAMP;
```

### 2. 批量插入
```sql
INSERT INTO revenue_data (date, merchant_name, game_name, currency, total_bet, total_payout, total_rounds, provider)
VALUES 
  ('2025-10-31', 'Betfarms', 'Bank Heist', 'USD', 121038.3167, 114553.5091, 575922, 'GMP'),
  ('2025-10-31', 'Betfarms', 'Chicken Road', 'USD', 1866526.2172, 1799867.5686, 4020044, 'GMP')
ON DUPLICATE KEY UPDATE 
  total_bet = VALUES(total_bet),
  total_payout = VALUES(total_payout),
  total_rounds = VALUES(total_rounds),
  updated_at = CURRENT_TIMESTAMP;
```

## 八、数据维护建议

### 1. 定期清理旧数据
```sql
-- 删除90天前的数据
DELETE FROM revenue_data WHERE date < DATE_SUB(CURDATE(), INTERVAL 90 DAY);
DELETE FROM game_user_data WHERE date < DATE_SUB(CURDATE(), INTERVAL 90 DAY);
DELETE FROM merchant_user_data WHERE date < DATE_SUB(CURDATE(), INTERVAL 90 DAY);
DELETE FROM game_retention_data WHERE date < DATE_SUB(CURDATE(), INTERVAL 90 DAY);
DELETE FROM merchant_retention_data WHERE date < DATE_SUB(CURDATE(), INTERVAL 90 DAY);
```

### 2. 数据归档
如果数据量很大，可以考虑：
- 按月分区表
- 将历史数据归档到归档表
- 使用数据压缩

### 3. 性能优化
```sql
-- 定期分析表以更新统计信息
ANALYZE TABLE revenue_data;
ANALYZE TABLE game_user_data;
ANALYZE TABLE merchant_user_data;
ANALYZE TABLE game_retention_data;
ANALYZE TABLE merchant_retention_data;
```

## 九、注意事项

1. **日期格式**：使用 `DATE` 类型，格式为 `YYYY-MM-DD`
2. **金额精度**：使用 `DECIMAL(20,4)` 确保精度
3. **唯一性约束**：使用唯一索引防止重复数据
4. **计算字段**：使用 `GENERATED ALWAYS AS` 自动计算，减少存储空间
5. **字符集**：使用 `utf8mb4` 支持特殊字符
6. **时区**：确保数据库时区设置正确
