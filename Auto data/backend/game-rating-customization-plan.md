# 游戏评级报告定制化优化方案

## 一、当前状态
目前的评级报告采用统一的模板和分析框架，所有游戏使用相同的分析角度和建议方向。

## 二、定制化维度

### 1. 根据游戏类型定制（推荐优先级：⭐⭐⭐⭐⭐）

#### 游戏类型识别
通过 `game_code` 前缀识别：
- `gp_crash_*` → Crash类游戏（如Aero Rush）
- `gp_lottery_*` → 彩票类游戏（如Golazo Win）
- `gp_slot_*` → 老虎机类游戏
- `gp_classic_*` → 经典类游戏（如Mines）

#### 定制化建议

**Crash类游戏（crash）**：
- **重点指标**：更关注 D7留存（长期复玩），新户下注占比（首充转化）
- **分析角度**：强调游戏机制的成瘾性和复玩节奏
- **改进建议模板**：
  - 优化首充门槛（降低第一局门槛）
  - 强化留存活动（连续游戏奖励）
  - 调整倍数曲线（平衡刺激度与派彩率）

**彩票类游戏（lottery）**：
- **重点指标**：更关注新户下注占比（规模），人均GGR（价值）
- **分析角度**：强调开奖频率和奖池吸引力
- **改进建议模板**：
  - 优化开奖频率（提高用户参与度）
  - 设置奖池活动（吸引新用户）
  - 调整派彩率（平衡收益与吸引力）

**老虎机类游戏（slot）**：
- **重点指标**：更关注 D1留存（短期体验），派彩下注比（风险控制）
- **分析角度**：强调画面吸引力和奖励机制
- **改进建议模板**：
  - 优化首充体验（新手引导）
  - 调整中奖频率（提升短期留存）
  - 平衡派彩率（控制风险）

**经典类游戏（classic，如Mines）**：
- **重点指标**：更关注 D7留存（核心用户），人均GGR（长期价值）
- **分析角度**：强调核心玩法和策略深度
- **改进建议模板**：
  - 优化新手教程（降低上手门槛）
  - 增加策略深度（提升复玩价值）
  - 设置长期任务（提升留存）

### 2. 根据评级等级定制（推荐优先级：⭐⭐⭐⭐）

**S级游戏（≥80分）**：
- **分析重点**：优势放大、规模化策略、标杆案例
- **建议模板**：重点分析成功因素，提出规模化放量建议，打造标杆案例

**A级游戏（65-79分）**：
- **分析重点**：优势巩固、短板补强、适度放量
- **建议模板**：巩固优势指标，针对性改进不足，可适度放量但需持续监控

**B级游戏（50-64分）**：
- **分析重点**：核心问题诊断、快速优化、谨慎放量
- **建议模板**：聚焦核心问题（留存/价值），提出快速优化方案，需解决主要问题后再考虑放量

**C级游戏（<50分）**：
- **分析重点**：紧急止损、深度优化、停止投放
- **建议模板**：立即停投，深度分析失败原因（玩法/数值/渠道），提出全面优化方案

### 3. 根据数据特征定制（推荐优先级：⭐⭐⭐）

**高留存低价值（D7高，GGR低）**：
- 分析：用户喜欢游戏但消费意愿低
- 建议：优化付费点设计，提升人均价值

**低留存高价值（D7低，GGR高）**：
- 分析：用户愿意付费但留存差
- 建议：优化游戏体验，提升复玩意愿

**小样本问题（新用户数<1000）**：
- 分析：数据量不足，需谨慎评估
- 建议：扩大样本量后再做决策

**红色渠道集中（大部分平台为C级）**：
- 分析：可能是游戏本身问题
- 建议：优先优化游戏本身，而非渠道

### 4. 根据平台币种定制（推荐优先级：⭐⭐）

**不同币种区域特点**：
- MXN（墨西哥）：关注本地化推广策略
- THB（泰国）：关注文化适配
- BRL（巴西）：关注支付方式优化
- USDT：国际化策略

## 三、实现方案

### 方案A：在AI提示词中注入游戏类型上下文（推荐）

**优点**：实现简单，只需修改提示词
**缺点**：依赖AI理解能力，可能不够精确

**实现步骤**：
1. 在 `game-rating-calculator.js` 中识别游戏类型
2. 在输入JSON中添加 `game_type` 字段
3. 在 `ai-rating-report-prompt.md` 中添加针对不同类型游戏的指导

### 方案B：动态生成分析建议模板（推荐）

**优点**：更精确，可控性强
**缺点**：需要维护模板库

**实现步骤**：
1. 创建 `game-rating-templates.js` 存储不同类型/等级的模板
2. 在生成报告前，根据游戏类型和等级选择模板
3. AI提示词中注入选定的模板作为参考

### 方案C：混合方案（最推荐）

**优点**：结合两者优势，既有AI灵活性又有模板精确性
**缺点**：实现稍复杂

**实现步骤**：
1. 在 `game-rating-calculator.js` 中添加游戏类型识别和特征分析
2. 输出增强的JSON，包含：
   ```json
   {
     "game": {...},
     "game_type": "crash",  // crash/lottery/slot/classic
     "performance_profile": "high_retention_low_value",  // 数据特征
     "customization": {
       "focus_metrics": ["d7_retention", "new_user_bet_ratio"],
       "analysis_angle": "强调游戏机制的成瘾性和复玩节奏",
       "suggestion_template": "crash_high_retention"
     }
   }
   ```
3. 在AI提示词中使用这些定制化信息指导分析

## 四、建议实施优先级

1. **第一阶段**（1-2天）：
   - 实现游戏类型识别
   - 在JSON中注入游戏类型
   - 在AI提示词中添加类型相关指导

2. **第二阶段**（3-5天）：
   - 实现评级等级定制化建议
   - 添加数据特征识别
   - 优化AI提示词结构

3. **第三阶段**（可选）：
   - 创建模板库
   - 实现平台币种定制化
   - 添加更细粒度的定制规则

## 五、示例：定制化后的提示词片段

```markdown
## 游戏类型：Crash类游戏

**重点分析维度**：
- D7留存率（crash游戏核心指标，反映长期复玩意愿）
- 新户下注占比（反映首充转化效果）
- 人均GGR（反映用户付费意愿）

**改进建议方向**：
- 优化首充门槛：降低第一局游戏门槛，提升新用户转化
- 强化留存活动：设置连续游戏奖励，提升D7留存
- 调整倍数曲线：平衡刺激度与派彩率，优化长期留存

## 评级等级：A级（65-79分）

**分析重点**：优势巩固 + 短板补强 + 适度放量

**预算建议**：可适度放量，但需持续监控关键指标（特别是D7留存和人均GGR）
```

## 六、预期效果

1. **更精准的分析**：针对不同类型游戏提供更符合其特点的分析
2. **更可执行的建议**：基于游戏类型给出具体可行的优化方向
3. **更高的报告价值**：运营团队可以更快速地理解问题和采取行动
4. **更好的决策支持**：不同等级游戏的预算决策更加明确
