# 调试输入数据结构使用说明

## 问题分析

从你的截图来看，问题在于：

### 1. 输入数据格式
你的输入数据是Lark API的响应格式：
```json
{
  "code": 0,
  "data": {
    "properties": {...},
    "sheets": [...],
    "spreadsheetToken": "..."
  },
  "msg": "success"
}
```

### 2. 缺少商户数据
输入数据中**没有包含实际的商户数据**，只有Lark表格的元数据信息。

### 3. 工作流问题
可能的问题：
- 上游节点没有正确输出商户数据
- 工作流连接配置错误
- 商户数据被包装在API响应中，需要正确提取

## 调试步骤

### 1. 使用调试代码
将 `backend/debug-input-data.js` 的代码复制到你的"创建Lark子表"节点中。

### 2. 运行调试
执行工作流，查看输出结果，了解：
- 输入数据的结构
- 是否包含商户数据
- 数据来源和格式

### 3. 分析结果
根据调试输出确定问题所在：
- 如果显示"没有找到商户数据"，说明上游节点有问题
- 如果显示"发现商户数据"，说明数据格式需要调整

## 可能的问题和解决方案

### 问题1：上游节点没有输出商户数据
**症状**：调试显示"没有找到商户数据"
**解决方案**：
1. 检查上游节点是否正确执行
2. 确认上游节点输出了商户数据
3. 检查工作流连接

### 问题2：商户数据被包装在API响应中
**症状**：调试显示"发现商户数据"，但格式不匹配
**解决方案**：
1. 修改代码来正确提取API响应中的数据
2. 使用 `data.data` 来访问嵌套的数据

### 问题3：工作流连接错误
**症状**：调试显示输入数据不是预期的格式
**解决方案**：
1. 检查工作流连接
2. 确认数据流向
3. 重新配置节点连接
