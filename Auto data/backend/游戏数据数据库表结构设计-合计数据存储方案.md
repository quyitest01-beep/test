# 游戏数据数据库表结构设计 - 合计数据存储方案

## 问题

如果 `date` 字段是 `DATE` 类型，那么"合计"这种汇总数据怎么存储？

## 解决方案

### 方案1：使用 NULL + period_range（推荐）✅

**设计思路**：
- `date` 字段允许为 `NULL`
- 当 `date` 为 `NULL` 时，表示这是汇总数据
- 添加 `period_range` 字段存储周期范围（如 "20251027-20251102"）
- 添加 `data_type` 字段区分数据类型（daily/weekly/monthly）

**表结构**：
```sql
CREATE TABLE `game_user_data` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT,
  `date` DATE DEFAULT NULL COMMENT '日期（NULL表示汇总数据）',
  `period_range` VARCHAR(50) DEFAULT NULL COMMENT '周期范围（汇总数据使用）',
  `data_type` ENUM('daily', 'weekly', 'monthly') DEFAULT 'daily',
  `game_name` VARCHAR(200) NOT NULL,
  `bet_users` INT UNSIGNED NOT NULL DEFAULT 0,
  ...
  UNIQUE KEY `uk_date_game` (`date`, `game_name`, `data_type`),
  UNIQUE KEY `uk_period_game_type` (`period_range`, `game_name`, `data_type`)
);
```

**数据示例**：
```sql
-- 日数据
INSERT INTO game_user_data (date, game_name, bet_users, data_type)
VALUES ('2025-10-31', 'Bank Heist', 1726, 'daily');

-- 周汇总数据（合计）
INSERT INTO game_user_data (date, period_range, game_name, bet_users, data_type)
VALUES (NULL, '20251027-20251102', 'Bank Heist', 1726, 'weekly');

-- 月汇总数据（合计）
INSERT INTO game_user_data (date, period_range, game_name, bet_users, data_type)
VALUES (NULL, '202511', 'Bank Heist', 5000, 'monthly');
```

**查询示例**：
```sql
-- 查询日数据
SELECT date, game_name, bet_users 
FROM game_user_data 
WHERE date IS NOT NULL 
  AND data_type = 'daily'
ORDER BY date, game_name;

-- 查询周汇总数据
SELECT period_range, game_name, bet_users 
FROM game_user_data 
WHERE date IS NULL 
  AND data_type = 'weekly'
ORDER BY period_range, game_name;
```

**优点**：
- ✅ 保持 `date` 为 DATE 类型，便于日期范围查询
- ✅ 清晰区分日数据和汇总数据
- ✅ 支持多种汇总类型（周/月）
- ✅ 索引优化，查询性能好

**缺点**：
- ⚠️ 需要处理 NULL 值
- ⚠️ 唯一索引需要分别处理日数据和汇总数据

---

### 方案2：使用特殊日期值

**设计思路**：
- 使用特殊日期值表示汇总数据，如 `0000-00-00` 或 `9999-12-31`
- 添加 `period_range` 字段存储周期范围

**表结构**：
```sql
CREATE TABLE `game_user_data` (
  `date` DATE NOT NULL COMMENT '日期（0000-00-00表示汇总数据）',
  `period_range` VARCHAR(50) DEFAULT NULL COMMENT '周期范围',
  ...
);
```

**数据示例**：
```sql
-- 日数据
INSERT INTO game_user_data (date, game_name, bet_users)
VALUES ('2025-10-31', 'Bank Heist', 1726);

-- 汇总数据
INSERT INTO game_user_data (date, period_range, game_name, bet_users)
VALUES ('0000-00-00', '20251027-20251102', 'Bank Heist', 1726);
```

**优点**：
- ✅ `date` 字段不为 NULL，查询简单
- ✅ 不需要处理 NULL 值

**缺点**：
- ❌ 使用特殊值不够直观
- ❌ 某些数据库可能不支持 `0000-00-00`
- ❌ 日期范围查询需要排除特殊值

---

### 方案3：使用单独的汇总表

**设计思路**：
- 日数据存储在 `game_user_data` 表
- 汇总数据存储在 `game_user_data_summary` 表

**表结构**：
```sql
-- 日数据表
CREATE TABLE `game_user_data` (
  `date` DATE NOT NULL,
  `game_name` VARCHAR(200) NOT NULL,
  `bet_users` INT UNSIGNED NOT NULL,
  ...
);

-- 汇总表
CREATE TABLE `game_user_data_summary` (
  `period_range` VARCHAR(50) NOT NULL,
  `data_type` ENUM('weekly', 'monthly') NOT NULL,
  `game_name` VARCHAR(200) NOT NULL,
  `bet_users` INT UNSIGNED NOT NULL,
  ...
);
```

**优点**：
- ✅ 表结构清晰，职责分离
- ✅ 查询逻辑简单

**缺点**：
- ❌ 需要维护两个表
- ❌ 查询时需要 UNION 合并数据
- ❌ 数据一致性需要额外保证

---

### 方案4：使用 VARCHAR 类型存储日期

**设计思路**：
- 将 `date` 字段改为 `VARCHAR` 类型
- 可以存储 "2025-10-31" 或 "合计"

**表结构**：
```sql
CREATE TABLE `game_user_data` (
  `date` VARCHAR(20) NOT NULL COMMENT '日期（"合计" 或 "YYYY-MM-DD"）',
  `game_name` VARCHAR(200) NOT NULL,
  `bet_users` INT UNSIGNED NOT NULL,
  ...
);
```

**优点**：
- ✅ 可以直接存储 "合计"
- ✅ 查询逻辑简单

**缺点**：
- ❌ 失去 DATE 类型的优势（日期范围查询、日期函数）
- ❌ 需要手动验证日期格式
- ❌ 日期排序可能有问题

---

## 推荐方案：方案1（NULL + period_range）

### 最终表结构

```sql
CREATE TABLE IF NOT EXISTS `game_user_data` (
  `id` BIGINT UNSIGNED NOT NULL AUTO_INCREMENT COMMENT '主键ID',
  `date` DATE DEFAULT NULL COMMENT '日期（NULL表示汇总数据）',
  `period_range` VARCHAR(50) DEFAULT NULL COMMENT '周期范围（汇总数据使用，格式：YYYYMMDD-YYYYMMDD 或 YYYYMM）',
  `data_type` ENUM('daily', 'weekly', 'monthly') DEFAULT 'daily' COMMENT '数据类型',
  `game_name` VARCHAR(200) NOT NULL COMMENT '游戏名',
  `bet_users` INT UNSIGNED NOT NULL DEFAULT 0 COMMENT '投注用户数',
  `provider` VARCHAR(50) DEFAULT NULL COMMENT '厂商（GMP/popular）',
  `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `updated_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  KEY `idx_date` (`date`),
  KEY `idx_period_range` (`period_range`),
  KEY `idx_data_type` (`data_type`),
  KEY `idx_game_name` (`game_name`),
  -- 唯一索引
  UNIQUE KEY `uk_date_game` (`date`, `game_name`, `data_type`),
  UNIQUE KEY `uk_period_game_type` (`period_range`, `game_name`, `data_type`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='游戏用户数据表';
```

### 数据插入示例

```sql
-- 日数据
INSERT INTO game_user_data (date, game_name, bet_users, data_type, provider)
VALUES ('2025-10-31', 'Bank Heist', 1726, 'daily', 'GMP')
ON DUPLICATE KEY UPDATE 
  bet_users = VALUES(bet_users),
  updated_at = CURRENT_TIMESTAMP;

-- 周汇总数据（合计）
INSERT INTO game_user_data (date, period_range, game_name, bet_users, data_type, provider)
VALUES (NULL, '20251027-20251102', 'Bank Heist', 1726, 'weekly', 'GMP')
ON DUPLICATE KEY UPDATE 
  bet_users = VALUES(bet_users),
  updated_at = CURRENT_TIMESTAMP;

-- 月汇总数据（合计）
INSERT INTO game_user_data (date, period_range, game_name, bet_users, data_type, provider)
VALUES (NULL, '202511', 'Bank Heist', 5000, 'monthly', 'GMP')
ON DUPLICATE KEY UPDATE 
  bet_users = VALUES(bet_users),
  updated_at = CURRENT_TIMESTAMP;
```

### 查询示例

```sql
-- 查询日数据
SELECT date, game_name, bet_users 
FROM game_user_data 
WHERE date IS NOT NULL 
  AND data_type = 'daily'
  AND date BETWEEN '2025-10-01' AND '2025-10-31'
ORDER BY date, game_name;

-- 查询周汇总数据
SELECT period_range, game_name, bet_users 
FROM game_user_data 
WHERE date IS NULL 
  AND data_type = 'weekly'
  AND period_range = '20251027-20251102'
ORDER BY game_name;

-- 查询所有数据（日数据 + 汇总数据）
SELECT 
  COALESCE(date, period_range) as date_or_period,
  game_name, 
  bet_users,
  data_type
FROM game_user_data 
WHERE game_name = 'Bank Heist'
ORDER BY 
  CASE WHEN date IS NOT NULL THEN 0 ELSE 1 END,
  date, 
  period_range;
```

### 约束条件

1. **日数据**：`date IS NOT NULL` 且 `period_range IS NULL`
2. **汇总数据**：`date IS NULL` 且 `period_range IS NOT NULL`
3. **唯一性**：
   - 日数据：`date + game_name + data_type` 唯一
   - 汇总数据：`period_range + game_name + data_type` 唯一

### 应用层处理

在应用层（n8n Code节点）中，可以这样处理：

```javascript

{
  date: '2025-10-31',

}

// 周汇总数据（合计）
{
  date: null,

  game_name: 'Bank Heist',
  bet_users: 1726,
  data_type: 'weekly'
}
```

## 总结

**推荐使用方案1**，因为：
1. ✅ 保持 DATE 类型的优势
2. ✅ 清晰区分日数据和汇总数据
3. ✅ 支持多种汇总类型
4. ✅ 索引优化，查询性能好
5. ✅ 符合数据库设计规范
