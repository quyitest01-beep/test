# 业务数据报告生成系统使用说明

## 系统概述

本系统用于自动化生成游戏平台的业务数据报告，支持周度和月度统计，包含新游戏、商户、游戏、币种、留存等多个维度的深度分析。

## 文件说明

### 1. Athena SQL查询语句
**文件**: `athena-query-business-report.sql`

**功能**: 从Athena数据库中查询业务数据报告所需的所有数据

**使用方法**:
1. 替换SQL中的参数：
   - `:start_date` - 当前期开始日期（格式：YYYY-MM-DD）
   - `:end_date` - 当前期结束日期（格式：YYYY-MM-DD）
   - `:previous_start_date` - 上期开始日期（格式：YYYY-MM-DD）
   - `:previous_end_date` - 上期结束日期（格式：YYYY-MM-DD）
   - `:new_game_release_month` - 新游戏上线月份（格式：YYYY-MM）

2. 执行查询，获取结果数据

**注意事项**:
- 表名和字段名需要根据实际数据库结构调整
- 需要根据实际表结构添加JOIN条件
- 新游戏识别逻辑需要根据实际release_date字段调整
- 留存数据类型需要根据实际表结构调整

### 2. 数据清洗Code代码
**文件**: `data-cleaner-for-business-report.js`

**功能**: 处理SQL查询结果，格式化为AI可理解的结构化数据

**在n8n中使用**:
1. 创建Code节点
2. 将SQL查询结果作为输入
3. 复制代码内容到Code节点
4. 配置输出格式

**输出格式**:
```json
{
  "periodInfo": {
    "currentPeriod": "20251027-1102",
    "previousPeriod": "20251020-1026",
    "periodType": "weekly"
  },
  "overall": {...},
  "newGames": [...],
  "merchants": {...},
  "games": {...},
  "currencies": {...},
  "retention": {...}
}
```

### 3. AI提示词
**文件**: `business-report-ai-agent-prompt-new-requirements.txt`

**功能**: 指导AI Agent生成符合要求的业务数据报告

**在n8n中使用**:
1. 创建AI Agent节点
2. 在"System Message"字段中粘贴提示词内容
3. 在"Prompt (User Message)"字段中使用：`{{ $json }}`（传入清洗后的数据）
4. 配置Structured Output Parser，使用`ai-output-schema-business-report.json`

**关键要求**:
- AI必须使用输入数据中的真实数据
- 必须严格按照要求的章节结构和格式输出
- 不能编造任何数据

### 4. AI输出JSON Schema
**文件**: `ai-output-schema-business-report.json`

**功能**: 限制AI Agent的输出格式，确保输出符合要求

**在n8n中使用**:
1. 在AI Agent节点中启用"Require Specific Output Format"
2. 添加Structured Output Parser
3. 在Schema字段中粘贴JSON Schema内容

**输出格式**:
```json
{
  "markdown": "完整的Markdown格式报告内容",
  "reportType": "weekly|monthly",
  "currentPeriod": "20251027-1102",
  "previousPeriod": "20251020-1026",
  "metadata": {
    "timestamp": "2025-11-05T04:16:49.898Z",
    "hasNewGames": true
  }
}
```

### 5. HTML转换代码
**文件**: `business-report-json-to-html.js`

**功能**: 将AI Agent输出的JSON报告转换为格式化的HTML，用于PDF生成

**在n8n中使用**:
1. 创建Code节点
2. 将AI Agent的输出作为输入
3. 复制代码内容到Code节点
4. 输出HTML内容，可用于PDF生成

**输出格式**:
- HTML格式的完整报告
- 包含样式和格式
- 支持打印和PDF转换

## 工作流程

### 完整工作流程（n8n）

```
1. HTTP Request节点 / Code节点
   ↓
   [获取SQL查询结果]
   ↓
2. Code节点（数据清洗）
   ↓
   [格式化数据]
   ↓
3. AI Agent节点
   ↓
   [生成Markdown报告]
   ↓
4. Code节点（HTML转换）
   ↓
   [转换为HTML]
   ↓
5. HTTP Request节点 / Google Docs节点
   ↓
   [保存或发送报告]
```

### 详细步骤

#### 步骤1: 数据查询
- 使用Athena SQL查询或通过HTTP Request获取数据
- 确保数据包含所有必需的字段

#### 步骤2: 数据清洗
- 使用`data-cleaner-for-business-report.js`
- 输入：SQL查询结果
- 输出：格式化的JSON数据

#### 步骤3: AI分析
- 使用`business-report-ai-agent-prompt-new-requirements.txt`作为System Message
- 输入：清洗后的JSON数据
- 输出：包含markdown字段的JSON对象

#### 步骤4: HTML转换
- 使用`business-report-json-to-html.js`
- 输入：AI输出的JSON（包含markdown字段）
- 输出：HTML格式的报告

#### 步骤5: 保存/发送
- 将HTML保存为文件或发送到Google Docs
- 可选：转换为PDF

## 统计维度说明

### 新游戏数据
- **识别规则**: 分析周期对应月份内上线的游戏视为新游戏
- **分析指标**: 总GGR、投注额、派奖额、局数、投注用户数、RTP、次留、7留
- **Top 5排行**: 主要商户、主要币种（按GGR值排序）

### 商户维度
- **商户GGR**: 整体对比、Top 3增长/下滑商户
- **商户投注**: 整体对比、Top 3增长/下滑商户、Top 3人均投注增长/下滑商户

### 游戏维度
- **游戏GGR**: 整体对比、Top 3增长/下滑游戏
- **游戏投注**: 整体对比、Top 3增长/下滑游戏、Top 3人均投注增长/下滑游戏
- **游戏局数**: 整体对比、Top 3增长/下滑游戏
- **游戏RTP**: 整体对比、Top 3增长/下滑游戏

### 币种维度
- **整体GGR对比**: 当前期 vs 上期
- **Top 3增长/下滑币种**: 按环比变化排序

### 留存数据
- **TOP20新用户次日留存排行**（当日用户数≥50）
- **TOP20新用户7日留存排行**（当日用户数≥50）
- **TOP20活跃用户次日留存排行**（当日用户数≥50）
- **TOP20活跃用户7日留存排行**（当日用户数≥50）

## 数据格式要求

### 输入数据格式
- SQL查询结果需要包含所有必需的字段
- 数据清洗后的JSON需要符合指定的结构

### 输出数据格式
- AI输出的JSON必须包含`markdown`字段
- Markdown内容必须符合指定的章节结构
- 所有数值必须使用正确的格式（货币、百分比、大数字）

## 常见问题

### 1. AI输出虚假数据
**原因**: 
- 提示词不够明确
- 输入数据格式不正确
- AI没有正确提取数据

**解决方案**:
- 使用最新的提示词文件
- 确保输入数据格式正确
- 在提示词中多次强调必须使用真实数据

### 2. 数据格式错误
**原因**:
- SQL查询结果格式不匹配
- 数据清洗代码未正确处理

**解决方案**:
- 检查SQL查询结果格式
- 调整数据清洗代码以匹配实际数据格式

### 3. HTML样式问题
**原因**:
- CSS样式未正确应用
- 表格格式不正确

**解决方案**:
- 检查HTML转换代码
- 确保CSS样式正确包含

## 注意事项

1. **数据准确性**: 确保SQL查询结果准确，所有数据都来自真实数据源
2. **日期格式**: 确保日期格式统一，使用YYYY-MM-DD格式
3. **数值计算**: 确保环比计算正确，处理除零情况
4. **新游戏识别**: 确保新游戏识别逻辑正确，使用正确的月份判断
5. **币种代码过滤**: 确保商户维度分析中过滤掉币种代码（3个大写字母）
