# n8n数据处理工作流配置指南

## 🎯 整体流程架构

```
上游数据 → 数据拆分 → 获取映射 → 商户匹配 → 游戏匹配 → 数据合并 → 输出
```

## 📊 节点配置详解

### 1. **数据拆分节点**
- **节点类型**: Code
- **节点名称**: "拆分商户游戏数据"
- **代码**: 使用 `split-merchant-game-data.js`
- **模式**: "Run Once for All Items"
- **输出**: 两个分支（商户数据、游戏数据）

### 2. **获取映射数据节点**
- **节点类型**: Code  
- **节点名称**: "获取商户游戏映射"
- **代码**: 使用 `fetch-merchant-game-mapping.js`
- **模式**: "Run Once for All Items"
- **输出**: 商户映射和游戏映射数据

### 3. **商户ID匹配节点**
- **节点类型**: Code
- **节点名称**: "商户ID匹配"
- **代码**: 使用 `merchant-id-matching.js`
- **模式**: "Run Once for All Items"
- **输入**: 商户数据分支 + 映射数据
- **输出**: 带商户名的商户数据

### 4. **游戏ID匹配节点**
- **节点类型**: Code
- **节点名称**: "游戏ID匹配"
- **代码**: 使用 `game-id-matching.js`
- **模式**: "Run Once for All Items"
- **输入**: 游戏数据分支 + 映射数据
- **输出**: 带游戏名的游戏数据

### 5. **数据合并节点**
- **节点类型**: Code
- **节点名称**: "合并处理数据"
- **代码**: 使用 `merge-processed-data.js`
- **模式**: "Run Once for All Items"
- **输入**: 匹配后的商户数据 + 匹配后的游戏数据
- **输出**: 完整的处理结果

## 🔗 节点连接配置

### 连接顺序：
1. **上游数据** → **数据拆分**
2. **数据拆分** → **获取映射数据** (并行)
3. **数据拆分** → **商户ID匹配**
4. **数据拆分** → **游戏ID匹配**
5. **获取映射数据** → **商户ID匹配**
6. **获取映射数据** → **游戏ID匹配**
7. **商户ID匹配** → **数据合并**
8. **游戏ID匹配** → **数据合并**

## ⚙️ 关键配置要点

### 1. **数据拆分节点**
- 根据 `stat_type` 字段判断数据类型
- 输出两个分支：商户数据和游戏数据
- 处理上万行数据时注意性能

### 2. **映射数据获取**
- 可以从Lark表格API获取
- 可以从数据库查询获取
- 建议缓存映射数据以提高性能

### 3. **ID匹配节点**
- 使用映射表进行ID到名称的转换
- 处理未匹配的ID（标记为"未知"）
- 记录匹配状态和时间戳

### 4. **数据合并**
- 按时间字段排序
- 添加数据分类标识
- 生成处理摘要

## 🚀 性能优化建议

### 1. **大数据处理**
- 使用"Split In Batches"节点分批处理
- 每批处理1000-5000条记录
- 避免一次性处理上万条数据

### 2. **映射数据缓存**
- 将映射数据存储在n8n变量中
- 避免重复获取映射数据
- 定期更新映射数据

### 3. **错误处理**
- 添加错误处理节点
- 记录处理失败的记录
- 提供重试机制
