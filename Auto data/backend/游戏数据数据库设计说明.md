# 游戏数据数据库表结构设计说明

## 一、表结构概览

设计了3个核心表来存储游戏相关数据：

1. **game_mapping** - 游戏映射表
2. **game_activity** - 游戏活跃数据表（原始明细数据）
3. **game_activity_summary** - 游戏数据汇总表（最终输出格式）

## 二、表结构详细说明

### 1. game_mapping（游戏映射表）

**用途**：存储 game_id 到 game_name 的映射关系

**字段说明**：
- `id`: 主键，自增
- `game_id`: 游戏ID（唯一）
- `game_name`: 游戏名称
- `merchant_id`: 商户ID（可选，用于记录数据来源）
- `created_at`: 创建时间
- `updated_at`: 更新时间

**索引**：
- 主键：`id`
- 唯一索引：`uk_game_id`（确保 game_id 唯一）
- 普通索引：`idx_game_name`、`idx_merchant_id`

**数据示例**：
```sql
INSERT INTO game_mapping (game_id, game_name, merchant_id) VALUES
('1698217747704', 'Bank Heist', '1760338096'),
('1698217747804', 'Aero Rush', '1760338096');
```

### 2. game_activity（游戏活跃数据表）

**用途**：存储游戏每日/每周/每月的活跃用户原始数据

**字段说明**：
- `id`: 主键，自增
- `date_str`: 日期字符串（日数据使用，格式：YYYYMMDD）
- `month_str`: 月份字符串（月数据使用，格式：YYYYMM）
- `period_range`: 周期范围（周数据使用，格式：YYYYMMDD-YYYYMMDD）
- `merchant`: 商户标识
- `game_id`: 游戏ID
- `game_name`: 游戏名称（映射后的名称，如果未匹配则为game_id）
- `data_type`: 数据类型（game_daily / game_weekly / game_monthly）
- `daily_unique_users`: 每日活跃用户数
- `weekly_unique_users`: 每周活跃用户数
- `monthly_unique_users`: 每月活跃用户数
- `is_matched`: 是否匹配到游戏名（0=未匹配，1=已匹配）
- `match_type`: 匹配类型（game_id_to_name / game_id_not_found）
- `created_at`: 创建时间
- `updated_at`: 更新时间

**索引**：
- 主键：`id`
- 单列索引：`date_str`、`month_str`、`period_range`、`merchant`、`game_id`、`game_name`、`data_type`、`is_matched`
- 复合索引：`idx_date_game`、`idx_period_game`、`idx_month_game`（优化常用查询）

**数据示例**：
```sql
-- 日数据
INSERT INTO game_activity (date_str, merchant, game_id, game_name, data_type, daily_unique_users, is_matched, match_type)
VALUES ('20251031', '1760338096', '1698217747804', '1698217747804', 'game_daily', 85, 0, 'game_id_not_found');

-- 周数据
INSERT INTO game_activity (month_str, period_range, merchant, game_id, game_name, data_type, weekly_unique_users, is_matched, match_type)
VALUES ('202511', '20251027-20251102', '1760338096', '1698217747804', '1698217747804', 'game_weekly', 1, 0, 'game_id_not_found');

-- 月数据
INSERT INTO game_activity (month_str, merchant, game_id, game_name, data_type, monthly_unique_users, is_matched, match_type)
VALUES ('202511', '1760338096', '1698217747804', 'Aero Rush', 'game_monthly', 100, 1, 'game_id_to_name');
```

### 3. game_activity_summary（游戏数据汇总表）

**用途**：存储合并后的游戏数据汇总（最终输出格式），便于快速查询和报表生成

**字段说明**：
- `id`: 主键，自增
- `date`: 日期（"合计" 或 "YYYYMMDD"）
- `game_name`: 游戏名称（可能是game_id）
- `bet_users`: 投注用户数
- `data_type`: 数据类型（daily / weekly / monthly，可选）
- `period_range`: 周期范围（周/月数据使用）
- `created_at`: 创建时间
- `updated_at`: 更新时间

**索引**：
- 主键：`id`
- 单列索引：`date`、`game_name`、`data_type`
- 复合索引：`idx_date_game`（优化按日期和游戏查询）
- 唯一索引：`uk_date_game_type`（防止重复数据）

**数据示例**：
```sql
-- 日数据汇总
INSERT INTO game_activity_summary (date, game_name, bet_users, data_type)
VALUES ('20251031', 'Bank Heist', 1726, 'daily')
ON DUPLICATE KEY UPDATE bet_users = VALUES(bet_users), updated_at = CURRENT_TIMESTAMP;

-- 周数据汇总（合计）
INSERT INTO game_activity_summary (date, game_name, bet_users, data_type, period_range)
VALUES ('合计', '1698217747804', 1, 'weekly', '20251027-20251102')
ON DUPLICATE KEY UPDATE bet_users = VALUES(bet_users), updated_at = CURRENT_TIMESTAMP;
```

## 三、表关系图

```
game_mapping (游戏映射表)
    │
    │ game_id (1:N)
    │
    ▼
game_activity (游戏活跃数据表)
    │
    │ 汇总后
    │
    ▼
game_activity_summary (游戏数据汇总表)
```

## 四、常用查询示例

### 1. 查询某日期的所有游戏数据
```sql
SELECT date, game_name, bet_users 
FROM game_activity_summary 
WHERE date = '20251031' 
ORDER BY game_name;
```

### 2. 查询某个游戏的汇总数据
```sql
SELECT date, bet_users 
FROM game_activity_summary 
WHERE game_name = 'Bank Heist' 
ORDER BY date;
```

### 3. 查询未匹配的游戏ID
```sql
SELECT DISTINCT game_id, COUNT(*) as count 
FROM game_activity 
WHERE is_matched = 0 
GROUP BY game_id 
ORDER BY count DESC;
```

### 4. 查询某个周期内的游戏活跃数据
```sql
SELECT game_name, SUM(weekly_unique_users) as total_users
FROM game_activity
WHERE period_range = '20251027-20251102'
  AND data_type = 'game_weekly'
GROUP BY game_name
ORDER BY total_users DESC;
```

### 5. 查询某个游戏的日活跃趋势
```sql
SELECT date_str, daily_unique_users
FROM game_activity
WHERE game_name = 'Bank Heist'
  AND data_type = 'game_daily'
  AND date_str >= '20251001'
ORDER BY date_str;
```

### 6. 统计匹配率
```sql
SELECT 
  COUNT(*) as total,
  SUM(CASE WHEN is_matched = 1 THEN 1 ELSE 0 END) as matched,
  SUM(CASE WHEN is_matched = 0 THEN 1 ELSE 0 END) as unmatched,
  ROUND(SUM(CASE WHEN is_matched = 1 THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 2) as match_rate
FROM game_activity
WHERE created_at >= DATE_SUB(NOW(), INTERVAL 7 DAY);
```

## 五、数据插入建议

### 1. 批量插入游戏映射数据
```sql
INSERT INTO game_mapping (game_id, game_name, merchant_id) 
VALUES 
  ('1698217747704', 'Bank Heist', '1760338096'),
  ('1698217747804', 'Aero Rush', '1760338096')
ON DUPLICATE KEY UPDATE 
  game_name = VALUES(game_name), 
  updated_at = CURRENT_TIMESTAMP;
```

### 2. 批量插入活跃数据
```sql
INSERT INTO game_activity 
  (date_str, merchant, game_id, game_name, data_type, daily_unique_users, is_matched, match_type)
VALUES 
  ('20251031', '1760338096', '1698217747804', '1698217747804', 'game_daily', 85, 0, 'game_id_not_found'),
  ('20251031', '1760338096', '1698217747704', 'Bank Heist', 'game_daily', 1726, 1, 'game_id_to_name');
```

### 3. 批量插入汇总数据（使用 ON DUPLICATE KEY UPDATE 避免重复）
```sql
INSERT INTO game_activity_summary (date, game_name, bet_users, data_type, period_range)
VALUES 
  ('合计', '1698217747804', 1, 'weekly', '20251027-20251102'),
  ('20251031', 'Bank Heist', 1726, 'daily', NULL)
ON DUPLICATE KEY UPDATE 
  bet_users = VALUES(bet_users), 
  updated_at = CURRENT_TIMESTAMP;
```

## 六、数据维护建议

### 1. 定期清理旧数据
```sql
-- 删除30天前的活跃数据
DELETE FROM game_activity 
WHERE created_at < DATE_SUB(NOW(), INTERVAL 30 DAY);

-- 删除30天前的汇总数据
DELETE FROM game_activity_summary 
WHERE created_at < DATE_SUB(NOW(), INTERVAL 30 DAY);
```

### 2. 数据备份
建议定期备份 `game_mapping` 表（映射关系很重要）

### 3. 性能优化
- 如果数据量很大，可以考虑按日期分区
- 定期分析表以更新统计信息：`ANALYZE TABLE game_activity;`

## 七、扩展建议

### 1. 如果需要支持多商户的游戏名映射
可以修改 `game_mapping` 表，添加复合唯一索引：
```sql
ALTER TABLE game_mapping 
ADD UNIQUE KEY `uk_merchant_game` (`merchant_id`, `game_id`);
```

### 2. 如果需要记录数据来源
可以在 `game_activity` 表中添加 `source` 字段：
```sql
ALTER TABLE game_activity 
ADD COLUMN `source` VARCHAR(50) DEFAULT NULL COMMENT '数据来源';
```

### 3. 如果需要支持更多统计维度
可以添加 `game_activity_stats` 表，存储更详细的统计信息

## 八、注意事项

1. **字符集**：使用 `utf8mb4` 以支持emoji等特殊字符
2. **时区**：确保数据库时区设置正确
3. **唯一性**：使用 `ON DUPLICATE KEY UPDATE` 避免重复数据
4. **索引**：根据实际查询需求调整索引
5. **数据量**：如果数据量很大，考虑分区或归档策略
