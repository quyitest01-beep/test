# 商户数据格式化器使用指南

## 功能概述

这个商户数据格式化器将商户匹配后的数据转换为Lark表格格式，按商户名A→Z排序，生成合计和每日数据。

## 输入数据格式

### 商户匹配后的数据
```javascript
{
  "merchant": "12POKIES",           // 商户名（已匹配）
  "stat_type": "merchant_daily",   // 数据类型
  "daily_unique_users": "1525",    // 每日用户数
  "date_str": "20251013",          // 日期
  "isMatched": true,               // 匹配成功标记
  "merchant_id": "1698202251",     // 原始商户ID
  "main_merchant_name": "主商户名"  // 主商户名
}
```

## 输出数据格式

### Lark表格格式
```javascript
{
  "商户名": "12POKIES",
  "日期": "合计",
  "投注用户数": 1525
}
```

```javascript
{
  "商户名": "12POKIES", 
  "日期": "20251013",
  "投注用户数": 1525
}
```

## 数据处理逻辑

### 1. 数据收集
- 只处理 `isMatched: true` 的商户数据
- 按商户名分组收集数据
- 累计每个商户的总用户数

### 2. 排序规则
- **商户名排序**：按A→Z字母顺序排序
- **日期排序**：每个商户内的日期按时间顺序排序

### 3. 数据生成
- **合计行**：每个商户先输出合计行（日期="合计"）
- **每日数据**：合计行后输出该商户的每日数据

## 输出示例

```javascript
[
  {
    "商户名": "12POKIES",
    "日期": "合计", 
    "投注用户数": 1525
  },
  {
    "商户名": "12POKIES",
    "日期": "20251013",
    "投注用户数": 1525
  },
  {
    "商户名": "13POKIES",
    "日期": "合计",
    "投注用户数": 1420
  },
  {
    "商户名": "13POKIES", 
    "日期": "20251013",
    "投注用户数": 1420
  }
]
```

## 特性

1. **智能排序**：支持中英文混合的商户名排序
2. **数据聚合**：自动计算每个商户的总用户数
3. **格式统一**：输出标准的Lark表格格式
4. **结构清晰**：合计行在前，每日数据在后

## 使用场景

1. **商户数据展示**：将商户数据格式化后写入Lark表格
2. **数据报告生成**：生成结构化的商户数据报告
3. **数据导出**：将处理后的数据导出到其他系统

## 注意事项

1. 确保输入数据包含 `isMatched: true` 标记
2. 商户名必须是字符串类型
3. 用户数字段会自动转换为数字类型
4. 日期格式为 YYYYMMDD 字符串

现在你可以使用这个格式化器来生成符合Lark表格要求的商户数据了！
