# 游戏数据合并器修复说明

## 问题描述

### 问题1：`game=null` 的数据没有显示该 `game_id`

**现象**：输入数据中存在 `game: null` 的记录，但这些记录没有被输出。

**原因**：原代码在收集数据时检查 `item.game` 存在才处理，`null` 值被过滤掉了。

**修复**：
- 在收集数据时，如果 `game` 为 `null` 或 `undefined`，则使用 `game_id` 作为 `game` 的值
- 确保所有数据都被处理，即使是未匹配的游戏

**代码变更**：
```javascript
// 修复前
if (item && item.dataType && item.game && ...) {
  allData.push(item);
}

// 修复后
if (item && item.dataType && ...) {
  if (!item.game && item.game_id) {
    item.game = String(item.game_id);
    item.isMatched = false;
    item.matchType = 'game_id_fallback';
  }
  if (item.game) {
    allData.push(item);
  }
}
```

### 问题2：存在多余的合计数据

**现象**：每个游戏都有两条合计数据，例如 "Bank Heist" 有两条合计：5856 和 2825。

**原因**：
1. 同一游戏可能同时存在 `game_weekly` 和 `game_monthly` 两种类型的合计数据
2. 原代码的合并键包含了周期信息（`period_range` 或 `month_str`），导致不同周期的合计数据没有被合并
3. 输出时，每种类型的合计数据都被单独输出，导致同一游戏有多条合计

**修复**：
1. **合并键优化**：合计数据的合并键只包含 `game` 和 `dataType`，不包含周期信息
   - 修复前：`${period_range}_${game}_${dataType}`
   - 修复后：`${game}_${dataType}`

2. **输出优化**：每个游戏只输出一条合计数据，合并所有类型的合计数据（`game_weekly` + `game_monthly`）
   - 将所有合计数据的用户数累加
   - 只输出一条合计行

**代码变更**：
```javascript
// 修复前：合并键包含周期信息
const mergeKey = isTotal
  ? `${item.period_range || item.month_str || '合计'}_${gameKey}_${item.dataType}`
  : `${item.date_str || '合计'}_${item.dataType}_${gameKey}`;

// 修复后：合计数据合并键不包含周期信息
const mergeKey = isTotal
  ? `${gameKey}_${item.dataType}`  // 只区分游戏和类型
  : `${item.date_str || '合计'}_${item.dataType}_${gameKey}`;

// 修复前：每种类型的合计都输出
totals.forEach(t => {
  finalRows.push({
    日期: '合计',
    游戏名: gameName,
    投注用户数: users
  });
});

// 修复后：合并所有类型的合计，只输出一条
if (totals.length > 0) {
  let totalUsers = 0;
  totals.forEach(t => {
    const users = toInt(t.weekly_unique_users || t.monthly_unique_users || 0);
    totalUsers += users;
  });
  finalRows.push({
    日期: '合计',
    游戏名: gameName,
    投注用户数: totalUsers
  });
}
```

## 注意事项

1. **合计数据合并**：如果同一游戏同时存在 `game_weekly` 和 `game_monthly`，它们会被合并成一条合计数据，用户数为两者的总和。

2. **`game=null` 处理**：所有 `game=null` 的记录都会使用 `game_id` 作为 `游戏名` 输出，确保数据不丢失。

3. **数据完整性**：修复后的代码确保所有输入数据都被处理，不会因为 `game=null` 而丢失数据。

3. 运行工作流，检查输出是否符合预期
