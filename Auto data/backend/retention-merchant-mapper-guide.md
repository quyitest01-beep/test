# 留存数据商户映射器使用指南

## 功能说明
这个Code节点专门处理留存数据的商户映射，将商户ID转换为商户名，并支持动态处理14日和30日留存数据。

## 输入数据格式
支持以下数据格式：

### 1. 商户映射数据
```json
{
  "sub_merchant_name": "betfiery",
  "main_merchant_name": "RD1", 
  "merchant_id": 1698202251
}
```

### 2. 新用户留存数据
```json
{
  "merchant": "1716179958",
  "new_date": "2025-10-15",
  "d0_users": "5824",
  "d1_users": "696", 
  "d1_retention_rate": "11.95",
  "d3_users": "387",
  "d3_retention_rate": "6.64",
  "d7_users": "0",
  "d7_retention_rate": "0.0",
  "dataType": "merchant_new"
}
```

### 3. 活跃用户留存数据
```json
{
  "merchant": "1760338096",
  "cohort_date": "2025-10-19",
  "d0_users": "32",
  "d1_users": "0",
  "d1_retention_rate": "0.0",
  "d3_users": "0", 
  "d3_retention_rate": "0.0",
  "d7_users": "0",
  "d7_retention_rate": "0.0",
  "dataType": "merchant_act"
}
```

## 输出数据格式
```json
{
  "商户名": "betfiery",
  "日期": "2025-10-15",
  "数据类型": "新用户留存",
  "当日用户数": 5824,
  "次日用户数": 696,
  "次日留存率": 11.95,
  "3日用户数": 387,
  "3日留存率": 6.64,
  "7日用户数": 0,
  "7日留存率": 0.0,
  "14日用户数": 123,      // 仅当上游数据存在时
  "14日留存率": 2.11,     // 仅当上游数据存在时
  "30日用户数": 45,       // 仅当上游数据存在时
  "30日留存率": 0.77,     // 仅当上游数据存在时
  "original_merchant_id": "1716179958",
  "main_merchant_name": "RD1",
  "isFullyMatched": true
}
```

## 主要特性

### 1. 智能数据识别
- 自动识别商户映射数据（`filtered_merchants` 数组）
- 自动识别新用户留存数据（`dataType: merchant_new` 或 `game_new`）
- 自动识别活跃用户留存数据（`dataType: merchant_act` 或 `game_act`）

### 2. 动态留存指标处理
- **基础指标**：当日、次日、3日、7日用户数和留存率
- **扩展指标**：14日、30日用户数和留存率（仅当上游数据存在时添加）

### 3. 商户映射
- 将商户ID转换为商户名
- 保留主商户名信息
- 提供映射成功率统计

### 4. 数据排序
- 按商户名A→Z排序
- 按日期排序（新用户留存用`new_date`，活跃用户留存用`cohort_date`）

## 使用场景
1. **新用户留存分析**：分析新注册用户的留存情况
2. **活跃用户留存分析**：分析现有用户的活跃度留存
3. **商户维度留存**：按商户分析用户留存情况
4. **游戏维度留存**：按游戏分析用户留存情况

## 注意事项
1. 确保上游数据包含商户映射信息
2. 留存数据必须包含`merchant`字段
3. 日期字段根据数据类型使用`new_date`或`cohort_date`
4. 14日和30日留存数据为可选，仅当上游数据存在时才会输出
