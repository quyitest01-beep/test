# 留存数据映射器修复说明

## 问题描述

**现象**：当游戏映射失败（找不到对应的游戏名）时，`game` 字段为 `null`，导致这些数据无法正常处理和显示。

**需求**：如果没有匹配的游戏名，应该使用游戏ID作为游戏名，确保所有数据都能正常处理。

## 修复内容

### 1. 游戏映射逻辑修复

**修复前**：
```javascript
const gameName = gameId ? gameIdToNameMap.get(gameId) : null;
const mappedData = {
  game: gameName || data.game_id,  // 如果gameName为null，使用data.game_id
  ...
};
```

**修复后**：
```javascript
const gameId = data.game_id ? data.game_id.toString() : null;
const gameName = gameId ? gameIdToNameMap.get(gameId) : null;
// 修复：如果游戏名未匹配，使用游戏ID作为游戏名
const finalGameName = gameName || gameId;

const mappedData = {
  game: finalGameName,              // 使用游戏名或游戏ID
  game_id: gameId,                  // 保留原始游戏ID
  game_matched: !!gameName,         // 游戏是否映射成功
  game_name_used: finalGameName,    // 实际使用的游戏名（可能是游戏名或游戏ID）
  ...
};
```

### 2. 数据收集和分组修复

**修复前**：
```javascript
// 只处理映射成功的游戏
const sortedGameNames = Array.from(new Set(matchedResults
  .filter(item => item.json.game_matched)  // 只包含映射成功的
  .map(item => item.json.game)
)).sort(...);
```

**修复后**：
```javascript
// 获取所有游戏名（包括使用游戏ID的情况）
const allGameNames = Array.from(new Set(matchedResults.map(item => 
  item.json.game_name_used || item.json.game
))).filter(Boolean);

// 按游戏名A→Z排序（游戏ID通常是数字字符串，会排在前面）
const sortedGameNames = allGameNames.sort((a, b) => {
  return a.localeCompare(b, 'zh-CN', { numeric: true });
});
```

### 3. 数据过滤修复

**修复前**：
```javascript
const gameData = matchedResults
  .filter(item => item.json.game_matched && item.json.game === gameName)
  .map(item => item.json);
```

**修复后**：
```javascript
// 获取该游戏的所有数据（使用 game_name_used 字段，包含游戏ID的情况）
const gameData = matchedResults
  .filter(item => {
    const itemGameName = item.json.game_name_used || item.json.game;
    return itemGameName === gameName;
  })
  .map(item => item.json);
```

### 4. 统计信息增强

**新增统计**：
```javascript
// 统计使用游戏ID的数据
const gameIdUsedCount = finalResults.filter(item => {
  const gameName = item.json.游戏名;
  // 检查是否是纯数字字符串（游戏ID）
  return /^\d+$/.test(gameName);
}).length;
console.log(`📊 使用游戏ID作为游戏名的数据行数: ${gameIdUsedCount}`);
```

## 修复效果

### 修复前

- 游戏映射失败时，`game` 字段为 `null`
- 这些数据可能无法正常处理和显示
- 最终输出中可能缺少这些数据

### 修复后

- 游戏映射失败时，`game` 字段使用游戏ID
- 所有数据都能正常处理和显示
- 最终输出中，未匹配的游戏使用游戏ID作为游戏名

**示例输出**：
```json
[
  {
    "游戏名": "Bank Heist",      // 映射成功的游戏
    "商户名": "Betfarms",
    "日期": "20251027",
    ...
  },
  {
    "游戏名": "1698217747804",   // 映射失败，使用游戏ID
    "商户名": "mexlucky",
    "日期": "20251027",
    ...
  }
]
```

## 关键字段说明

- **`game`**：游戏名或游戏ID（映射成功使用游戏名，失败使用游戏ID）
- **`game_id`**：原始游戏ID（始终保留）
- **`game_matched`**：布尔值，表示是否成功映射到游戏名
- **`game_name_used`**：实际使用的游戏名（可能是游戏名或游戏ID），用于分组和排序

## 注意事项

1. **游戏ID格式**：游戏ID通常是纯数字字符串（如 `"1698217747804"`），可以通过正则 `/^\d+$/` 识别
2. **排序顺序**：游戏ID（数字字符串）通常会排在游戏名（中英文）前面
3. **数据完整性**：修复后确保所有留存数据都被处理，不会因为映射失败而丢失

## 使用说明

1. 将修复后的代码复制到 n8n 的 Code 节点
2. 确保上游数据包含 `game_id` 字段
3. 运行工作流，检查输出
4. 查看控制台日志，确认使用游戏ID的数据行数
