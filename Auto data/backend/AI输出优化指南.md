## 使用建议

### 在n8n工作流中的配置

1. **数据预处理节点**：
   ```
   Merge13.json → 数据预处理 → 业务数据分析器 → AI Agent
   ```

2. **AI Agent节点配置**：
   - **System Message**：使用 `business-report-ai-agent-prompt.md` 的内容
   - **Prompt**：简要说明分析任务
   - **输入数据**：确保传入Merge13.json格式的数据

3. **后处理验证节点**（可选）：
   ```
   AI Agent → ai-output-validator.js → 转换为HTML
   ```

4. **HTML转换节点**：
   ```
   AI输出 → ai-report-to-html.js → HTML报告
   ```

### 关键配置点

1. **确保AI Agent接收到正确的数据格式**：
   - 第一个元素：新游戏列表
   - 后续元素：明细数据记录（包含 `period` 或 `is_previous` 字段）

2. **Prompt中强调的重点**：
   - 必须从原始明细数据中提取新游戏的商户和币种排行
   - 必须生成指定格式的Markdown列表
   - 不要输出重复标题

3. **验证输出质量**：
   - 使用 `ai-output-validator.js` 检查输出
   - 查看验证报告，确认所有必需信息都已包含

## 常见问题排查

### Q1: AI输出没有新游戏商户/币种排行

**原因**：
- AI可能没有正确提取新游戏记录
- 数据字段名不匹配

**解决方案**：
1. 检查输入数据是否包含新游戏列表
2. 检查新游戏记录是否包含商户和币种字段
3. 在Prompt中强调必须提取这些数据
4. 提供更详细的提取逻辑示例

### Q2: 标题格式不正确

**原因**：
- 标题提取逻辑不匹配数据格式
- 周期信息格式不统一

**解决方案**：
1. 使用 `ai-report-to-html.js` 的 `extractTitle` 函数自动提取
2. 确保Markdown中包含明确的周期信息（格式：`2025-10-27 至 2025-11-02`）

### Q3: 表格显示不正确

**原因**：
- Markdown表格格式不规范
- 缺少表头分隔行

**解决方案**：
1. 确保表格包含表头分隔行（`|---|---|`）
2. 使用 `ai-report-to-html.js` 的表格处理函数
3. 检查表格单元格格式是否正确
