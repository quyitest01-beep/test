# 🧠 RAG增强查询系统指南

## 📋 **系统概述**

这是一个基于**RAG (检索增强生成)** 的智能查询系统，通过建立查数场景知识库，让AI能够检索相关文档来生成准确的SQL查询。

## 🎯 **核心优势**

- ✅ **智能检索** - AI可以检索相关文档获取上下文
- ✅ **准确SQL** - 基于知识库生成符合规范的SQL
- ✅ **场景覆盖** - 覆盖所有常见的查数场景
- ✅ **持续学习** - 可以不断更新知识库内容

## 🔧 **系统架构**

### **数据准备阶段**
```
知识库文档 → 嵌入向量 → 向量数据库
```

### **查询处理阶段**
```
用户消息 → AI Agent → 检索相关文档 → 生成增强SQL → 执行查询
```

## 🚀 **部署步骤**

### **步骤1: 设置知识库**

1. **导入知识库设置工作流**
   - 导入 `setup-knowledge-base.json`
   - 配置OpenAI凭证
   - 执行工作流设置知识库

2. **验证知识库设置**
   - 确认所有文档已成功插入
   - 检查嵌入向量是否正确生成

### **步骤2: 部署RAG增强查询系统**

1. **导入主工作流**
   - 导入 `rag-enhanced-query.json`
   - 配置OpenAI凭证
   - 配置Telegram Bot凭证

2. **配置AI Agent**
   - 设置Prompt包含检索文档上下文
   - 配置System Message包含数据库结构信息

3. **配置文档检索**
   - 设置Retrieve Documents节点
   - 配置Embedding Model
   - 设置检索数量限制

### **步骤3: 测试系统**

1. **发送测试消息**
   ```
   PANDA-1976423513265401856
   PANDA-1976422629802373120
   PANDA-1976437176340557824
   查询这些记录的详情
   ```

2. **验证检索效果**
   - 检查AI是否检索到相关文档
   - 验证生成的SQL是否准确
   - 确认查询结果格式正确

## 📚 **知识库内容**

### **核心查询场景**

1. **ID查询场景**
   - 处理具体记录ID查询
   - 支持多种ID格式
   - 生成标准SELECT语句

2. **用户查询场景**
   - 处理用户ID查询
   - 支持时间范围过滤
   - 按时间排序结果

3. **时间范围查询场景**
   - 处理时间段查询
   - 自动时间格式转换
   - 默认最近7天范围

4. **统计查询场景**
   - 处理汇总统计查询
   - 计算胜率、总额等
   - 使用聚合函数

5. **游戏查询场景**
   - 处理特定游戏查询
   - 支持游戏代码过滤
   - 按游戏类型分类

6. **商户查询场景**
   - 处理商户数据查询
   - 支持商户ID过滤
   - 按商户分组统计

### **数据库结构信息**

- **字段说明** - 详细的字段类型和格式
- **字段处理** - 时间戳转换、金额格式化
- **查询模式** - 默认条件、排序规则
- **优化建议** - 性能优化、成本控制

## 🔍 **工作流程详解**

### **1. 用户消息接收**
```
Telegram Trigger → 接收用户消息
```

### **2. 文档检索**
```
用户消息 → Retrieve Documents → 检索相关文档
```

### **3. AI分析增强**
```
原始消息 + 检索文档 → AI Agent → 生成增强分析
```

### **4. 智能路由**
```
分析结果 → Can Execute Check → 路由到不同处理分支
```

### **5. 查询执行**
```
可执行 → Execute Query → 执行SQL查询
```

### **6. 结果处理**
```
查询结果 → Format Result → 格式化输出
```

## 💡 **使用示例**

### **示例1: ID查询**
**用户输入**:
```
PANDA-1976423513265401856
PANDA-1976422629802373120
PANDA-1976437176340557824
查询这些记录的详情
```

**系统处理**:
1. 检索ID查询相关文档
2. 提取ID列表
3. 生成标准SELECT语句
4. 执行查询并返回结果

### **示例2: 用户查询**
**用户输入**:
```
查询用户 User9911684466 最近7天的游戏记录
```

**系统处理**:
1. 检索用户查询相关文档
2. 提取用户ID和时间范围
3. 生成带时间过滤的SQL
4. 执行查询并返回结果

### **示例3: 统计查询**
**用户输入**:
```
统计今天的交易数量和总金额
```

**系统处理**:
1. 检索统计查询相关文档
2. 识别统计需求
3. 生成聚合查询SQL
4. 执行查询并返回统计结果
