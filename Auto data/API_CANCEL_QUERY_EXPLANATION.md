# 取消查询 API 的作用说明

## 🎯 这个 API 是做什么的？

这个 API (`/api/query/cancel/:queryId`) 用于**取消正在执行的 Athena 查询**。

## 📋 主要作用

### 1. **停止长时间运行的查询** ⏹️

**场景**：你启动了一个查询，但发现：
- 查询写错了（SQL 有误）
- 查询会扫描太多数据（成本太高）
- 查询运行时间太长（超过预期）

**解决方案**：使用取消 API 立即停止查询

```javascript
// 在 n8n 中调用
POST /api/query/cancel/e6cc8574-bd83-403c-8234-be64a3e5b19f
```

### 2. **节省 AWS 成本** 💰

**问题**：Athena 按扫描的数据量收费（每 TB $5）
- 如果查询写错了，可能会扫描大量不必要的数据
- 长时间运行的查询会持续消耗资源

**解决**：及时取消错误查询，避免浪费

**示例**：
```
错误查询可能扫描：100 GB 数据 = $0.50
及时取消后：只扫描了 1 GB = $0.005
节省：$0.495
```

### 3. **释放系统资源** 🔄

**场景**：
- 系统有查询数量限制
- 内存或连接数有限
- 需要让其他查询优先执行

**解决**：取消不必要的查询，释放资源给重要查询

### 4. **用户体验优化** ⚡

**场景**：
- 用户在界面上启动了查询
- 发现查询条件不对
- 想要重新输入条件

**解决**：提供"取消"按钮，让用户可以立即停止并重新开始

## 🔐 API_KEY 的作用

### 为什么需要 API_KEY？

```
┌─────────┐
│  n8n    │  →  [需要认证] →  ┌──────────┐
│ 工作流  │                   │ 后端 API │
└─────────┘                   └──────────┘
```

**API_KEY 的作用**：

1. **身份验证** 🔒
   - 确保只有授权的人/服务可以取消查询
   - 防止恶意用户随意取消他人的查询

2. **安全保护** 🛡️
   - 防止未授权访问
   - 保护你的 API 不被滥用

3. **访问控制** 🎫
   - 可以给不同团队分配不同的 API_KEY
   - 便于追踪谁在使用 API

```
用户触发查询

    ↓
用户发现查询条件错误

调用取消 API ← 使用 API_KEY 认证
    ↓
查询立即停止

用户可以重新启动正确的查询
```

### 场景 2: 定时任务监控

```
定时任务启动大量查询
    ↓

    ↓
自动调用取消 API
    ↓
避免成本浪费
```

### 场景 3: 用户界面

```
用户在 Web 界面执行查询
    ↓
```

## 🔄 完整流程

### 1. 启动查询

```javascript
// 启动查询
POST /api/query/sql
{
  "sql": "SELECT * FROM large_table WHERE ..."
}

// 返回
{
  "queryId": "e6cc8574-bd83-403c-8234-be64a3e5b19f",
  "status": "running"
}
```

### 2. 检查查询状态

```javascript
// 检查状态
GET /api/query/status/e6cc8574-bd83-403c-8234-be64a3e5b19f

// 返回
{
  "status": "running",
  "dataScanned": "50 GB",
  "estimatedCost": "$0.25"
}
```

### 3. 发现需要取消

用户发现：
- 查询已经扫描了 50 GB（成本 $0.25）
- 预计还要扫描 200 GB（额外成本 $1.00）
- 查询条件有误，需要重新执行

### 4. 取消查询

```javascript
// 取消查询（需要 API_KEY）
POST /api/query/cancel/e6cc8574-bd83-403c-8234-be64a3e5b19f
Headers: {
  "X-API-Key": "your-api-key-here"
}

// 返回
{
  "success": true,
  "message": "Query cancelled successfully"
}
```

### 5. 验证取消

```javascript
// 再次检查状态
GET /api/query/status/e6cc8574-bd83-403c-8234-be64a3e5b19f

// 返回
{
  "status": "cancelled",
  "dataScanned": "50 GB",  // 只扫描了 50 GB
  "finalCost": "$0.25"      // 最终成本 $0.25（节省了 $1.00）
}
```
