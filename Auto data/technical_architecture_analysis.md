# 智能查询系统技术架构分析

## 第一阶段：后台系统开发（当前阶段）

### 核心模块组成

#### 1. 自然语言处理模块 (sqlGenerator.js)
- **功能**：将自然语言查询转换为SQL语句
- **实现机制**：
  - 基于关键词匹配的意图识别算法
  - 5种查询模板：销售、地区、客户、产品、时间分析
  - 自动参数提取（时间范围、数量限制）
  - SQL优化和安全验证

#### 2. 查询执行引擎 (athenaService.js) 
- **功能**：执行SQL查询并管理查询生命周期
- **集成服务**：AWS Athena
- **特性**：
  - 查询状态监控和取消
  - 结果获取和分页处理
  - 成本控制和超时管理

#### 3. Python代码生成器 (pythonCodeGenerator.js)
- **功能**：为复杂查询生成Python执行代码
- **支持**：数据分片和并行处理

#### 4. API路由层 (query.js)
- **RESTful接口**：
  - `/generate-sql` - SQL生成
  - `/execute` - 查询执行  
  - `/python-query` - Python查询
  - `/python-env` - 环境检查

#### 5. 导出服务模块 (exportService.js)
- **功能**：大规模数据分包导出Excel
- **解决痛点**：突破传统Excel数据量限制

### 模块协作流程

```
用户输入 → 前端页面 → API路由层 → 相应服务模块 → 数据库 → 返回结果
```

**标准查询流程**：
1. 用户输入自然语言 → 调用 `/generate-sql`
2. SQLGenerator分析意图 → 生成优化SQL
3. 调用 `/execute` → AthenaService执行查询
4. 返回结果给前端展示

**Python查询流程**：
1. 用户输入查询 → 调用 `/python-query`
2. PythonCodeGenerator生成代码 → PythonExecutor执行
3. 处理大数据分片 → 返回聚合结果

### 技术实现特点

1. **前后端分离架构**
   - 前端：React + Vite + Ant Design
   - 后端：Node.js + Express + AWS SDK

2. **弹性扩展设计**
   - 支持从简单查询到复杂大数据处理
   - 模块化服务架构，易于扩展

3. **安全可靠机制**
   - SQL注入防护
   - 查询成本控制
   - 完善的错误处理

## 第二阶段：自动化查询功能

### 新增模块

#### 1. 定时任务调度器
- **功能**：配置和管理定时查询任务
- **技术选型**：node-cron或类似调度库

#### 2. 报告生成器
- **功能**：自动生成查询结果报告
- **格式支持**：Excel、PDF、HTML

#### 3. 通知服务模块
- **功能**：多渠道发送查询报告
- **支持渠道**：电子邮件、Telegram、飞书

### 协作流程

```
定时触发器 → 任务调度器 → 执行查询 → 报告生成 → 通知发送
```

## 第三阶段：智能客服集成

### 新增模块

#### 1. 机器人接口层
- **功能**：处理机器人平台Webhook
- **支持平台**：Telegram、飞书、其他IM工具

#### 2. 自然语言理解模块
- **功能**：增强的意图识别和对话管理
- **技术方案**：集成NLP服务或自建模型

#### 3. 会话状态管理
- **功能**：维护多轮对话上下文
- **实现**：Redis或内存存储会话状态

### 协作流程

```
用户消息 → 机器人平台 → Webhook → 自然语言理解 → 查询执行 → 结果返回
```

## 技术架构优势

1. **渐进式开发**：三个阶段功能逐步叠加，降低开发风险
2. **模块化设计**：各功能模块职责单一，便于维护和测试
3. **扩展性强**：支持多种数据源和输出渠道
4. **用户体验优**：从命令行到图形界面到自然语言交互
