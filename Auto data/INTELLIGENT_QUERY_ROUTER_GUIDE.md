# 🧠 智能查询路由系统指南

## 📋 **系统概述**

这是一个智能查询路由系统，可以根据用户消息的复杂程度和完整性，自动选择最合适的处理方式：

```
用户消息 → AI分析 → 智能路由 → 自动执行/澄清问题/转人工
```

## 🎯 **三层处理机制**

### **1. 自动执行查询** ✅
**条件**: 置信度 ≥ 0.7 且 canExecute = true
**处理**: 直接生成SQL并执行查询

**支持的场景**:
- 明确的ID查询: `1976423513265401856 查询详情`
- 清晰的用户查询: `查询用户 li-57ebcc16aa1240a4bc9114578a4646ce 的记录`
- 明确的时间查询: `查询最近7天的数据`
- 基础统计查询: `统计今天的交易数量`

### **2. 智能澄清问题** 🤔
**条件**: 置信度 0.3-0.6 或 canExecute = false
**处理**: 生成澄清问题，引导用户提供更多信息

**典型场景**:
- 信息不完整: `查询某个用户的记录` (缺少具体用户ID)
- 条件模糊: `查询游戏数据` (缺少具体游戏或时间范围)
- 需要确认: `查询昨天的交易` (需要确认具体日期和条件)

### **3. 转人工处理** 🔧
**条件**: 置信度 < 0.3 或查询类型为"无法识别"
**处理**: 生成专业的人工转接消息

**典型场景**:
- 过于复杂的业务逻辑
- 多表关联查询
- 特殊的分析需求
- 无法理解的查询意图

## 🧠 **AI 分析能力**

### **支持的查询类型**
1. **ID查询** - 识别16位以上数字ID
2. **用户查询** - 识别uid、用户ID等
3. **时间查询** - 识别日期、时间相关词汇
4. **统计查询** - 识别数量、金额、统计等
5. **游戏查询** - 识别游戏代码、游戏类型
6. **商户查询** - 识别merchant、商户等
7. **混合查询** - 包含多种查询类型
8. **无法识别** - 超出系统理解范围

### **置信度评分系统**
- **0.9-1.0**: 信息完整，可直接执行
- **0.7-0.8**: 信息基本完整，可尝试执行
- **0.5-0.6**: 信息不完整，需要澄清
- **0.3-0.4**: 信息模糊，需要更多细节
- **0.0-0.2**: 无法理解，需要人工协助

## 🔄 **工作流程**

```
1. 用户发送消息到 Telegram 群组
   ↓
2. Query Analyzer 分析消息内容和意图
   ↓
3. Parse Analysis 解析 AI 分析结果
   ↓
4. Can Execute Check 判断处理方式
   ↓
5a. 自动执行查询 (置信度 ≥ 0.7)
   ↓
6a. 生成SQL → 执行查询 → 格式化结果 → 发送回复
   ↓
5b. 智能澄清 (置信度 0.3-0.6)
   ↓
6b. 生成澄清问题 → 发送回复
   ↓
5c. 转人工处理 (置信度 < 0.3)
   ↓
6c. 生成人工转接消息 → 发送回复
```
