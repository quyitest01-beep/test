# 账单发送失败重试工具使用指南

## 概述

`retry_failed_bills.py` 是一个专门用于重新发送失败账单的工具，可以避免重复发送给已成功的商户，支持多种使用模式。

## 功能特性

- ✅ **智能识别**：自动分析批次记录，识别发送成功和失败的商户
- 🔄 **多种重试模式**：支持自动、手动和交互式重试
- 📊 **详细分析**：提供发送状态的详细分析报告
- 💾 **结果记录**：自动保存重试结果，便于追踪
- 🎯 **精准重试**：只对失败商户重新发送，避免重复

## 使用模式

### 1. 自动模式（推荐）

基于批次ID自动识别失败商户并重试：

```bash
# 重试指定批次的失败商户
python retry_failed_bills.py --batch-id 20250101_120000

# 仅分析不重试
python retry_failed_bills.py --batch-id 20250101_120000 --analyze-only
```

**适用场景**：
- 刚完成一次批量发送，需要重试失败的商户
- 有明确的批次ID记录

### 2. 手动模式

手动指定需要重试的商户：

```bash
# 重试指定商户
python retry_failed_bills.py --manual --merchants "商户A,商户B,商户C"
```

**适用场景**：
- 明确知道哪些商户需要重试
- 需要对特定商户进行单独处理

### 3. 交互模式

通过交互界面选择批次和操作：

```bash
# 启动交互模式
python retry_failed_bills.py --interactive
```

**适用场景**：
- 不确定批次ID
- 需要查看可用的批次记录
- 希望在重试前确认操作

## 详细使用步骤

### 步骤1：确认环境

确保以下文件和目录存在：
- `complete_invoice_pdfs/` - PDF文件目录
- `unified_merchants_YYYYMM.json` - 商户配置文件
- `records/` - 批次记录目录（可选）

### 步骤2：选择合适的模式

#### 自动模式详细步骤

1. **查找批次ID**：
   ```bash
   # 查看可用的批次记录
   ls records/batch_*.json
   ```

2. **分析发送状态**：
   ```bash
   # 先分析不重试
   python retry_failed_bills.py --batch-id 20250101_120000 --analyze-only
   ```

3. **执行重试**：
   ```bash
   # 确认无误后执行重试
   python retry_failed_bills.py --batch-id 20250101_120000
   ```

#### 交互模式详细步骤

1. **启动交互模式**：
   ```bash
   python retry_failed_bills.py --interactive
   ```

2. **选择批次**：
   - 系统会显示可用的批次记录
   - 输入编号或直接输入批次ID

3. **确认重试**：
   - 查看分析结果
   - 确认是否执行重试

### 步骤3：查看结果

重试完成后，系统会：
- 在终端显示详细的重试结果
- 保存重试记录到 `records/retry_批次ID_时间戳.json`

## 故障排除

### 常见问题

#### 1. 批次记录文件不存在

**错误信息**：`[ERROR] 批次记录文件不存在`

**解决方案**：
- 检查 `records/` 目录是否存在
- 确认批次ID格式正确（如：20250101_120000）
- 使用手动模式指定商户

#### 2. PDF文件不存在

**错误信息**：`[ERROR] 未找到商户 XXX 的PDF文件`

**解决方案**：
- 检查 `complete_invoice_pdfs/` 目录
- 确认PDF文件命名格式：`商户名_YYYYMM_Bill.pdf`
- 重新生成缺失的PDF文件

#### 3. 商户配置缺失

**错误信息**：`[ERROR] 未找到主商户报告文件`

**解决方案**：
- 确认存在 `unified_merchants_YYYYMM.json` 文件
- 检查文件格式和内容是否正确
- 重新生成商户配置文件

#### 4. 发送失败

**可能原因**：
- 网络连接问题
- 邮箱或TG配置错误
- 目标地址无效

**解决方案**：
- 检查网络连接
- 验证邮箱和TG配置
- 手动测试发送功能

### 调试技巧

1. **使用分析模式**：
   ```bash
   python retry_failed_bills.py --batch-id XXXXX --analyze-only
   ```

2. **检查日志输出**：
   - 关注 `[ERROR]` 和 `[WARNING]` 信息
   - 查看具体的失败原因

3. **手动验证**：
   - 检查PDF文件是否存在且完整
   - 验证商户配置是否正确

## 最佳实践

### 1. 重试前的准备

- ✅ 确认网络连接正常
- ✅ 检查PDF文件完整性
- ✅ 验证商户配置准确性
- ✅ 先使用分析模式查看状态

### 2. 重试策略

- 🎯 **优先使用自动模式**：基于批次记录自动识别
- 🔍 **分析后再重试**：先用 `--analyze-only` 查看状态
- 📝 **记录重试原因**：在重试记录中注明重试原因
- ⏰ **避免频繁重试**：给失败的通道一些恢复时间

### 3. 结果验证

- 📊 查看重试统计信息
- 📁 检查保存的重试记录文件
- ✉️ 确认重要商户的发送状态
- 📞 必要时电话确认关键商户

### 4. 记录管理

- 🗂️ 定期清理旧的重试记录
- 📋 维护重试操作日志
- 🔄 跟踪重复失败的商户
- 📈 分析失败模式和改进点

## 高级用法

### 批量处理多个批次

```bash
# 处理多个批次（需要脚本支持）
for batch in 20250101_120000 20250101_140000 20250101_160000; do
    echo "处理批次: $batch"
    python retry_failed_bills.py --batch-id $batch
done
```

### 结合其他工具

```bash
# 生成重试报告
python retry_failed_bills.py --batch-id 20250101_120000 --analyze-only > retry_analysis.txt

# 重试后发送通知
python retry_failed_bills.py --batch-id 20250101_120000 && echo "重试完成" | mail -s "重试结果" admin@company.com
```
