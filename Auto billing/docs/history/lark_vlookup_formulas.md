## 🔧 VLOOKUP公式实现

### 1. 子商户数量公式（I列）
```
=COUNTIFS(表1!H:H, A2, 表1!C:C, "正常", 表1!D:D, "生产")
```

**公式说明**：
- `表1!H:H`：表1的主商户列
- `A2`：当前行的主商户名称
- `表1!C:C, "正常"`：状态为"正常"
- `表1!D:D, "生产"`：环境为"生产"

### 2. 子商户列表公式（J列）
```
=TEXTJOIN(", ", TRUE, IF(AND(表1!H:H=A2, 表1!C:C="正常", 表1!D:D="生产"), 表1!A:A, ""))
```

**公式说明**：
- `TEXTJOIN(", ", TRUE, ...)`：用逗号连接文本，忽略空值
- `AND(表1!H:H=A2, 表1!C:C="正常", 表1!D:D="生产")`：条件判断
- `表1!A:A`：子商户名称列

### 3. 主商户下拉选择（数据验证）
```
=UNIQUE(表1!H:H)
```

**设置步骤**：
1. 选择表2的A列
2. 数据 → 数据验证
3. 条件：列表
4. 来源：`=UNIQUE(表1!H:H)`

## 🔍 常见问题解决

### 1. 公式不更新
**问题**：修改表1数据后，表2公式结果不更新
**解决**：
- 检查公式语法是否正确
- 确保表1和表2在同一工作簿中
- 手动刷新或重新计算

### 2. 子商户列表为空
**问题**：子商户数量正确，但列表为空
**解决**：
- 检查TEXTJOIN函数语法
- 确认条件判断是否正确
- 验证子商户名称列是否有数据

### 3. 性能问题
**问题**：数据量大时公式计算缓慢
**解决**：
- 使用更精确的列范围（如H2:H1000）
- 避免在公式中使用整列引用
- 考虑使用数据透视表
