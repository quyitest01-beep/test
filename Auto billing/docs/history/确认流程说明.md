# 账单确认流程说明

## 🔄 当前流程（点击确认按钮后）

### 1. 确认按钮点击
- 邮件中的确认按钮链接到：`http://localhost:8000/billing/confirm/{confirmation_id}?action=confirm`
- 系统调用 `process_confirmation()` 方法

### 2. 自动触发客户发送
- ✅ **确认**：自动调用 `_send_customer_bills()` 发送账单给所有客户
- ❌ **拒绝**：不发送，只记录拒绝状态

### 3. 客户账单发送
- 系统会尝试获取每个商户的客户联系方式
- 发送邮件和Telegram通知给真实客户

## 🧪 测试模式配置

### 启用测试模式
```json
{
  "test_mode": true,
  "test_recipients": {
    "email": ["poon@gaming-panda.com"],
    "telegram": []
  },
  "override_customer_contacts": true
}
```

### 测试模式效果
- ✅ 所有客户账单都发送到测试邮箱 `poon@gaming-panda.com`
- ✅ 不会发送给真实客户
- ✅ 可以安全测试完整流程

## 🚀 使用步骤

### 1. 启用测试模式
```bash
# test_mode_config.json 已创建，test_mode: true
```

### 2. 启动确认服务器
```bash
python simple_confirmation_server.py
```

### 3. 发送内部确认邮件
```bash
python test_real_billing.py
```

### 4. 点击邮件中的确认按钮
- 浏览器会打开确认页面
- 显示处理结果
- 在测试模式下，客户账单发送到测试邮箱

### 5. 检查测试邮箱
- 查看 `poon@gaming-panda.com` 收到的客户账单

## ⚙️ 生产环境配置

### 禁用测试模式
```json
{
  "test_mode": false,
  "override_customer_contacts": false
}
```

### 生产环境效果
- 账单发送给真实客户邮箱和Telegram群
- 使用真实的客户联系方式
