## 📝 具体修改内容

### 1. 数据来源
- **主商户数据**：从 `master_merchant_report_*.json` 的 `mapped_data` 字段获取
- **子商户明细**：从每个主商户的 `sub_merchants[].currencies[].charge_usdt` 聚合
currency：邮件附件里的货币
total_prize：邮件附件里的总派奖
total_bet：邮件附件里的总投注
GGR = total_bet - total_prize
Fee Rate：对应主商户的费率（百分比）
usd_rate：邮件附件里的USD汇率
Charge = GGR × usd_rate × (fee_rate / 100)
應付縂金額 (USDT)：所有子商户的Charge之和
GGR (USDT)：所有子商户的GGR之和

### 2. 表格样式
- **列数**：2列（子商户名称 + 应付金额）
- **排序**：按金额降序排列（金额大的在前）
- **表头**：蓝色背景，白色文字
- **数据行**：左对齐（子商户名），右对齐（金额）

### 3. 显示逻辑
```
如果 有子商户明细数据：
    显示"子商户明细表"
    包含：子商户名称 + 聚合后的USDT金额
否则：
    显示原来的"主商户汇总表"
```

## ❓ 常见问题

**Q: 如果某个主商户没有子商户明细怎么办？**
A: 系统会自动回退到显示主商户汇总表

**Q: 子商户金额如何计算？**
A: 从 `currencies[].charge_usdt` 字段聚合，按子商户名称分组累加

**Q: 表格样式可以调整吗？**
A: 可以，在 `create_complete_data_table` 方法中修改样式设置
